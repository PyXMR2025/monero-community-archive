# Discussion History
## erciccione | 2020-04-15T07:20:44+00:00
Wouldn't be better to fetch the hashes from the repository instead? https://raw.githubusercontent.com/monero-project/monero-site/master/downloads/hashes.txt.

It's easier to spot a compromised file on GitHub. Maybe github could be the primary source and getmonero a fallback option in case github is down?

## selsta | 2020-04-15T07:22:05+00:00
We require the signature of two different maintainers, it does not matter for the updater if getmonero.org would be hacked.

## xiphon | 2020-04-15T11:08:33+00:00
This and also:

users doing manual update will use getmonero.org.

If getmonero.org gets hacked, there will be appropriate errors in the GUI console/logs, so we will react way faster on a hack.

If we use github as a source, we lose this opportunity.
