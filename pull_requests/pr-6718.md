# Original Description
Originally when the rolling median has an even number of elements stored, it returns the average of the middle two elements like this:
`return (a+b)/2`
However, if `(a + b)` is greater than the size of the type of `a` and `b`, the result might wrap around (e.g. if it's an integral type, etc.).

While it is unlikely wraparound will ever occur based on how Monero uses the rolling median, it's better to have safe code in case this template gets reused elsewhere. The new equation is
`(a+b)/2 = (a/2) + (b/2) + ((a - 2*(a/2)) + (b - 2*(b/2)))/2`

This is intended to play nice with integral division, however floating point types may have more rounding errors due to the additional operations.
