## iamamyth | 2020-11-09T23:19:12+00:00
This could be restricted a bit further:
```
^(([^:]+)://)?...
```

The `*?` makes no sense in this context both semantically (url schemes must contain at least one character, hence the change to `+`) and logically (there's a negated one-character class followed by the character being negated, so a greedy match is the same as a non-greedy one; in both cases, the match must end at the `:`, meaning `*?` can be replaced with `*`).
