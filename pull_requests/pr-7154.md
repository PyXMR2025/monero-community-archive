## SChernykh | 2020-12-17T17:43:42+00:00
It's really hard to understand this condition which is the key part of this PR. Why is it `block_in_chain != h` in the end?

## moneromooo-monero | 2020-12-17T17:59:46+00:00
We expect a set of new hashes from context.m_expect_height. We also expect an overlapping hash right before it, which is the point where the new hashes attach. Anything we get below that (I'm not sure it's valid to get anything below it right now, but let's assume so for the argument) cannot match the hash we have, or the new chain would have grafted higher. For the hash at the attach point, it must match what we have.
This can technically trigger false positives if there was a reorg between the query and the reply.
