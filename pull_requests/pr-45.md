# Original Description
## What
`helper/` is the kitchen-sink crate for very generic, not necessarily Cuprate specific functions, types, etc.

If a 3rd party's crate/functions/types are small enough, it could be moved here to trim dependencies and allow easy modifications.

## Currently Implemented
`helper` is split into modules - each one filling in some niche. Each module is feature gated with `#[cfg()]`, pulls in dependencies only when needed, and is `#[no_std]` if possible. All features/modules are enabled by default.

A list of the modules and some examples of what they're used for.

| Module   | Example function/type |
|----------|-----------------------|
| `asynch` | `InstaFuture<T>` - future that is ready immediately
| `atomic` | `AtomicF(32\|64)` - atomic float backed by `AtomicU(32\|64)`
| `crypto` | Cryptographic code - `check_point()`
| `num`    | Generic number code - `median<T>(nums: &[T]) -> T` where `T` is any number
| `thread` | Thread code - `THREADS()` for system thread count
| `time`   | System time and general time utils - `current_time()` for current UNIX timestamp
