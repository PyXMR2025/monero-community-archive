# Original Description
A POC of wrapping boost::serialization and boost::archive. \
\
The point here is to group together parts, which are meant to be compiled together: the boost serialization headers and the serialization of the classes itself. This ensures, that the serialization, and most importantly, the boost headers, aren't compiled at every incremental change of the serialized classes. This reduces the compilation time of these parts by about 20 seconds for each large file. Naturally the 20 seconds have to be spent in the wrappers, but these parts will be recompiled rarely. \
\
An additional side effect is grouping and wrapping the serialization in a central place, allowing to easily switch the underlying library at a later point. \
\
Also changed the header inclusion order in some files. Reference:
https://stackoverflow.com/questions/2762568/c-c-include-header-file-order
