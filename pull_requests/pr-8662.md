## vtnerd | 2022-12-26T19:07:16+00:00
Adding more to the discussion - the compiler is good at optimizing, but every `init_gens()` call produces a read-acquire statement that requires synchronization across cores. As an example, use gcc godbolt to generate code for a function-local static - the x86-64 ASM looks like a simple double-check lock but the arm64 ASM has a `dmb` instruction (memory barrier). This isn't a huge impact on performance, but does add stalls/wait cycles.

Constexpr was expanded in C++14, so compile-time conversion and 2's complement checking should now be possible across platforms:
```c++
template<typename T>
constexpr T bytes_to(const std::initializer_list<unsigned char> in)
{
    T out{}; // zero-initialize trailing bytes

    auto current = std::begin(out.data);
    if (std::end(out.data) - current < in.size())
        throw std::runtime_error{"invalid initializer size"};

    for (const auto byte : in)
        *current++ = byte;
    return out;
}

struct char_test { char data[5]; };
constexpr const auto test_case = bytes_to<char_test>({0x00, 0x7f, 0x80, 0xff});
static_assert(test_case.data[0] == 0, "");
static_assert(test_case.data[1] == 127, "");
static_assert(test_case.data[2] == char(-128), "");
static_assert(test_case.data[3] == char(-1), "");
static_assert(test_case.data[4] == 0, "");

constexpr const auto G = bytes_to<public_key>(
    {0x58, 0x66, 0x66, 0x66, 0x66, 0x66, 0x66, 0x66, 0x66, 0x66, 0x66, 0x66, 0x66, 0x66, 0x66, 0x66, 
    0x66, 0x66, 0x66, 0x66, 0x66, 0x66, 0x66, 0x66, 0x66, 0x66, 0x66, 0x66, 0x66, 0x66, 0x66}
);
```
Tested against x86-64, arm64, and risc-v on gcc godbolt.
