# Original Description
### What
This PR:
- Makes changes to the base RPC types
- Adds a documentation macro the whole crate can use
- Adds the `default` and `free` (internal) module for default/conditional serde behavior
- Makes changes to the type generator macro
    - More maintenance friendly (sub-macros instead of duplicated branches)
    - Supports default field values
- Adds the `misc` module containing _every single_ misc type used in other request/response types

---

This is the 1st out of 4 PRs that implement types in `cuprate-rpc-types`.

There are 3 PRs built on-top of this one:
1. https://github.com/Cuprate/cuprate/pull/219
2. https://github.com/Cuprate/cuprate/pull/220
3. https://github.com/Cuprate/cuprate/pull/221

These respectively implement all:
- `/json_rpc` types
- `/*.bin` types
- Other JSON types

## hinto-janai | 2024-07-08T23:50:51+00:00
The type generator macro no longer has duplicated code, it only has 1 branch and uses sub-macros instead.

Since we got rid of the RPC access `Request` stuff, this macro _always_ takes `Request {}` as input now. No fields will create the `type Request = ()`, where fields will create a normal request.

Responses still use the RPC access syntax, so it's one of:
- `Request` - adds nothing (some requests only have `status`, so we need this)
- `ResponseBase` - adds `base: crate::base::ResponseBase`
- `AccessResponseBase` - adds `base: crate::base::AccessResponseBase`
