# Original Description
Changes:
- Separate mobile/light wallets in two categories:
  - Local sync wallets: don't share any key with a remote server.
  - Remote sync (lightweight) wallets: share the private view key with a third-party server.

# Discussion History

```suggestion
  localsyncinfo: These wallets connect to @remote-nodes and scan the @blockchain on your device, without sharing any key with the nodes. They provide higher privacy, but require more time synchronizing, since missing @blocks will be scanned only when the wallet is opened.
```

## erciccione | 2021-08-05T07:13:15+00:00
```suggestion
  remotesyncinfo: These wallets share your private @view-key (needed to see incoming transactions) with a remote server, which scans @blocks looking for your transactions. They are faster to use because they continuously scan the blockchain looking for transactions, but your privacy can be lessened if you don't control the remote server.
```
