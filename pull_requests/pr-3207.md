# Original Description
Addresses #3206 

The command previously assumed that the wallet can have at most one tx corresponding to the requested txid which is either incoming or outgoing. This assumption, however, doesn't hold when one moves some funds from one subaddress account to another within the same wallet; the same tx would look like an outgoing transfer to one account, while it'd look like an incoming transfer to another account.

This patch adds `subaddress_account` to the command request which defaults to 0.
