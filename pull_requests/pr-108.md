# Original Description
Changes the free functions in `cuprate_test_utils::data` to return `&'static` and the superset types `VerifiedBlockInformation, TransactionVerificationData` instead of `Block, Transaction`.

Needed in https://github.com/Cuprate/cuprate/pull/102 tests since `add_block()` needs a `VerifiedBlockInformation` (using a dummy `VerifiedBlockInformation` isn't ideal since I'd also like to test if all the inner data were added to the right tables).

The new `VerifiedBlockMap` and `VerifiedTxMap` just map `Block, Transaction` into the verified forms with some help from static input.

This is sorta hacky since:
1. Static input isn't verified
2. It's only semi-automatic (ideally we just provide a `block.bin` file and all fields get computed/verified)

But doing this involves pulling in `consensus/` and doing PoW hash calculation so... doing it manually (for now).
