# Original Description
As discovered in [#4674](https://github.com/monero-project/monero/issues/4674#issuecomment-433190600), if the wallet .keys file needs to be updated -- for example, if the password is changed -- it is rewritten in an unsafe manner: the existing file is truncated and then rewritten, so if the process were to crash in between these two steps, the keys would be lost.

This patch fixes that by using the same mechanism already used to update the history file: write the data to a new file, and rename it into place.
