# Original Description
In Monero it is not possible to know who is the sender of an incoming transaction, unless you create a different address for every new person you will receive from. Currently this requires the user to add a label for every new address created, and not everyone knows that this kind of control should be done in order to identify the sender.

In my opinion the address book functionality should be integrated with this control, to make things easier. Currently the address book only stores a (single) destination address for each contact, but it would be useful if it also stored a (single) receiving address for each contact. This way the wallet would know that this receiving address was given only to that person.

This would be specially useful in GUI, allowing things like:
- in history tab: displaying a contact as the sender of all incoming transactions in a specific address
- in address book tab: displaying all transactions received from a contact in the address book page
- in address book tab: automatically generating a new receiving address for every new contact that is added
- in address book tab: selecting the receiving address of a contact directly in address book, instead of having to search the contact's address in receive page
- in receive page: add a section "receive from contact", where the user select a contact, and then his/her receiving address/QR code is displayed (without the user having to search the list of receiving addresses that were already generated by the wallet)
