# Original Description
Clang 10.0 [does not recognize](https://phabricator.services.mozilla.com/D87052) the `aarch64-apple-darwin` target. The release branch hacks around this by automatically [renaming the target](https://github.com/monero-project/monero/blob/release-v0.18/contrib/depends/hosts/darwin.mk#L3-L7).

This approach is less straightforward on master due to the [introduction of CMake](https://github.com/monero-project/monero/pull/9640) in depends. We would also need to inform CMake about the updated target to prevent it from passing `--target=aarch64-apple-darwin` to the compiler.

It's less messy to just rename the target to whatever clang expects.
