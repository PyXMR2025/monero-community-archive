# Original Description
Protect node privacy by proper filtering in restricted-mode RPC answers
    
This patch allows to filter out sensitive information for queries that rely on the pool state, when running in restricted mode. This filtering is only applied to data sent back to RPC queries. Results of inline commands typed locally in the daemon are not affected.

In practice, when running with `--restricted-rpc`:
* get_transaction_pool will list relayed transactions with the fields "last relayed time" and "received time" set to * get_transaction_pool will not list unrelayed transaction, and will not list key images that are used purely for unrelayed * get_transaction_pool_hashes.bin will not list unrelayed transaction
* get_transaction_pool_stats will not count unrelayed transactions in any of the aggregated values that are computed
* some other functions that at the moment do not have a corresponding call (for instance get_transaction_backlogs) also do not rely on unrelayed transactions if in restricted mode
    
The implementation does not make filtering the default, so developers should be careful to set the relevant parameter in the future when node privacy is required.
Fixes #2590.
