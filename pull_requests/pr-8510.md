# Original Description
PR of the code discussed in #8484 (the base patches in this PR are written by @moneromooo-monero). This PR should also patch #8476:

- Only offline wallets can import outputs.
- Empty exports won't corrupt wallet files when signing.
- Subaddress indexes are exported & imported.

I added explicit tests for the buggy failure cases described in #8476. But I wasn't able to reproduce @woodser's issue described in [this comment](https://github.com/monero-project/monero/issues/8484#issuecomment-1213632430):

> it occasionally hangs on import_outputs (when used correctly with view-only and offline wallets) for some unknown reason

Requesting @woodser check if he can repro the hang with this code; I suspect the hang might have something to do with the slightly off empty exports, which should be good with this PR.
