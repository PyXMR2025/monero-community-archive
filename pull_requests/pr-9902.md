# Original Description
This PR: If the client requests a `start_height` > current chain tip, then the daemon returns a successful response that includes the chain's height.

Current behavior on release is to ban a client that requests such a height.

This PR's behavior is useful both in this [FCMP++ integration PR's](https://github.com/seraphis-migration/monero/pull/81) approach to syncing that simplifies wallet refresh while minimizing data downloaded by the client, and in the async scanner.
