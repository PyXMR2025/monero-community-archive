# Original Description
Changes:
- removed trailing zeros in Fee

Changes in Received transations:
- added "In" field to the Blockheight field position. This field will display the receiving address index + label (from the receive page) or the truncated address (when the address has no label) (fixes #2987)
- Blockheight field moved to the bottom.
- increased height of collapsed received transactions to the same as sent transactions (320 pixels), in order to display the blockheight. As a side effect, transaction key is also being displayed for received transactions.

Changes in Sent transactions:
- moved "To" field to the Blockheight field position. This field will now display either an address book icon + a destination address label (from the address book) or a truncated destination address (when the destination address has no address book entry).
- Blockheight field moved to the bottom.
- changed "Waiting for transaction to leave txpool" message to "Waiting confirmation..." in order to fit in the smaller space and because it's easier to understand.
- transactions with amount = 0 XMR will display destination as "My wallet", since they are seen in self transactions after a wallet is rescanned or restored from keys/seed.
- in restored wallets, transactions sending to the same wallet will display "Unknown amount", since we can't recover the amount information from the blockchain
