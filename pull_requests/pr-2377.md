# Original Description
Addresses #2338. 

Due to the structure of the blockchain DB, outgoing payments with no change outputs (i.e. those created by `sweep_all`) can't be handled properly, and their spent heights are set to the current blockchain height (so that they appear at the bottom with `show_transfers`).

Also, unconfirmed (pending) outgoing payments are not handled in this patch (may be added later if necessary).
