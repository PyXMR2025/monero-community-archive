# Original Description
Currently, the `rescan_bc keep_ki` command silently fails. The assertion in `hash_m_transfers()` is incorrectly inverted, and always fails on the first call, when literal `-1` is passed. This exception isn't handled properly, so no message is shown.

This function also incorrectly treated some integer members of `struct transfer_details` as pointers, causing memory errors.

With these fixed, this command seems to provide at least a good workaround for  #7673.
