## hinto-janai | 2024-04-01T01:11:30+00:00
@Boog900 How do you think we should do retries on `WriteRequest`'s?

The current method of _reacting_ to resize errors instead of _proactively_ resizing comes with the downside in that we must `clone()` the data we use since we may not know if a resize is needed and the next `loop` iteration needs the data.

I propose we keep `RuntimeError::ResizeNeeded` but `DatabaseWriter` keeps track of the memory map byte size in some local `usize` counter, which we do book-keeping on each request:

1. `WriteRequest` received
2. Calculate how many bytes it'll write (with some buffer in-case?)
3. Check if our `current_memory_map_bytes: usize` needs to be resized
4. Resize if needed
5. Call write function
6. Panic if it errors with `ResizeNeeded` as we just handled that

Similar to: https://github.com/monero-project/monero/blob/c8214782fb2a769c57382a999eaf099691c836e7/src/blockchain_db/lmdb/db_lmdb.cpp#L665
