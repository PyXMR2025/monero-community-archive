# Original Description
The first patch which is about refactoring the use of settings in `cryptonote_config.h` was motivated by the second patch which is to allow the additional node addresses specified via flags like `--add-priority-node` to be resolved (the refactoring is used for obtaining the default P2P port).

I understand that the refactoring part may feel iffy especially because it touches `generate_genesis_block`. It can be dropped if it's deemed unnecessary and the second patch can be written without the refactoring.
