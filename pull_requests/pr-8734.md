# Original Description
This PR should not introduce any behavioral differences to the wallet code whatsoever. However, this PR simplifies connection handling and invocation in wallet2, and will allow for `NodeRPCProxy` to seamlessly switch out connections underneath `wallet2` without interrupting the wallet-y stuff. This is useful when the wallet wants to point to a "group" of potential daemons (think Electrum public nodes or the community node proposal #8624). 

Currently `wallet2` maintains a unique pointer to an abstract http client and `m_daemon_rpc_mutex` locks invokations to one-at-a-time. `wallet2` also maintains the "offline" status, daemon addresses, trusted/untrusted, login info, etc, as member fields. This PR moves that responsibility to `NodeRPCProxy`. Invokations are now also done through `NodeRPCProxy`.

The basic idea for future PRs will be to add a certain `tools::connection_settings` value that, when passed to `set_daemon`, will cause `NodeRPCProxy::get_transport_ref()` to reference a transport which manages connections to multiple different daemons and their penalty scores, not unlike `cryptonote::bootstrap_daemon`. 

Depends on #8724
