# Original Description
Hi all,

I am trying to run monero exclusively over tor in a VPS and cannot start because I am getting Address already in use error.

```bash

if [ -z "$ONION_HOST" ]; then
  echo "Error: Could not read onion hostname" >&2
  exit 1
fi

echo "Using onion address: $ONION_HOST"

torsocks monerod \
```

## nahuhh | 2025-07-24T09:44:48+00:00
- use `--proxy` instead of torsocks
- use rpc-restricted-bind-{port|ip} instead of changing rpc-bind-port
- _use correct port for anonymous-inbound_

```sh
#!/bin/sh

# Read real onion address
ONION_HOST=$(sudo cat /var/lib/tor/monero/hostname)
if [ -z "$ONION_HOST" ]; then
  echo "Error: Could not read onion hostname"
  exit 1
fi

echo "Using onion address: $ONION_HOST"

monerod \
  --no-zmq \
  --data-dir /mnt/102941054/monero/ \
  --prune-blockchain \
  --rpc-restricted-bind-ip 127.0.0.1 \
  --rpc-restricted-bind-port 18089 \
  --rpc-login "user:pass" \
  --proxy=127.0.0.1:9050 \
  --tx-proxy tor,127.0.0.1:9050,10 \
  --p2p-bind-port 18080 \
  --p2p-bind-ip 127.0.0.1 \
  --anonymous-inbound "$ONION_HOST:18084,127.0.0.1:18084,32" \
  --hide-my-port \
  --no-igd \
  --log-level 1
```

- here as well - use correct port for anonymous-inbound
```
HiddenServiceDir /var/lib/tor/monero/
HiddenServicePort 18084 127.0.0.1:18084   # Inbound tx-relay
HiddenServicePort 18089 127.0.0.1:18089   # RPC (for wallet, if needed)
```


Your problem was trying to bind anonymous-inbound to 18080. Monerod does not support p2p blockchain sync over onion. The port specified for anon-inbound is for inbound tx-relay amd must be a new port.
the above parameters will work
