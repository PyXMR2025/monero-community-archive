# Original Description
This PR is to implement all Carrot cryptography, as well as enote creation logic and enote scanning logic. This PR should exclude very Monero-specific integration details. The `carrot_core` library is largely meant to be a stand-alone library, and thus only includes headers from `src/crypto` and `src/ringct`. 

https://github.com/jeffro256/carrot/blob/master/carrot.md

- [x]  Non-legacy carrot enote creation
- [x]  Non-legacy carrot enote scanning
- [x] Tests for new enotes addressed to legacy cryptonote accounts/addresses
- [x] Transaction output set finalization logic
- [x] Efficient implementation for X25519 unclamped scalar multiplication
- [x] Hardware device interface
- [x] Switch commitments (walmart version)
- [x] Internal self-send messages
- [x] Fix Debian 10 build
- [x] Unbind the special janus anchor derivation from the account spend pubkey to make hybrid accounts easier
- [x] Rewrite ARM mx25519 implementation to support unclamped scalar multiplications
- [x] ~~Deterministic FCMP++ output rerandomizations to support some refund address schemes~~
- [x] ~~Low-dependency payment proposal to enote finalization C interface for hardware wallets~~
- [x] ~~Payment proofs to internal enotes which A) allow for burning bug validation and B) don't reveal $s_{vb}$~~
