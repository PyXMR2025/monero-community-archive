# Original Description
Solves [issue 7053](https://github.com/monero-project/monero/issues/7053).

**TL;DR: Cleaner device architecture will allow the hardware wallet vendors to integrate with Monero, without needing to merge their implementations into MoneroCore.**

Apologies for the PR being somewhat overweight.

## Problem
The the `libdevice` depends on `libcryptonote_basic` through `cryptonote::get_transaction_prefix_hash(cryptonote::transaction_prefix const&, crypto::hash&)`, used in `device_default.cpp`, while and `libcryptonote_basic` depends on `libdevice`, via the file `crypronote_basic.h`, where the `device.hpp` is included and used.

## Solution
Extracted `device_abstr` and using it whenever possible within the libraries. The implementation target `device` is only fed to the client applications.

## Extra work
I took the opportunity to remove the inclusion of the `device.hpp` from the `cryptonote_basic.hpp`, limiting the dependencies a bit. Compilation time reduction: about 3-4%. Compare with [master here](http://enjo.hopto.org/pub/monero/).

```

  Codegen & opts (backend):   1721.3 s
```


## Future work
This is out of scope for this PR, but Ideally, after only the client apps have the possibility to instantiate concrete devices, they should inject them into the library code via Abstract Factory. In our case, we wouldn't see a direct call to device creation anymore, like in `cryptonote_basic::account.cpp` through `hw::get_device`, which is a singleton, calling a concrete factory, but the function would receive the `AbstractDeviceFactory` via parameter from the client code, which instantiates a concrete factory, derived from `AbstractDeviceFactory`. This will allow for further abstraction of the client devices, not limiting Monero to concrete, predefined manufacturers in our own code.

```
src/cryptonote_basic/account.cpp:

void account_base::create_from_device(const std::string &device_name)
  {
    hw::device &hwdev =  hw::get_device(device_name);
    hwdev.set_name(device_name);
    create_from_device(hwdev);
  }
```
