## stoffu | 2019-01-04T00:43:18+00:00
Valid point, but the more I think, the more I find the idea of encrypted payment IDs being used for multi-destination txs nonsensical. Construction of such a tx is disabled officially anyway:

https://github.com/monero-project/monero/blob/e344d93ce706482fc55bb036051dc08da18fabbb/src/cryptonote_core/cryptonote_tx_utils.cpp#L236-L241

so we shouldn't need to worry about users relying on encrypted payment IDs being used in multi-destination txs. As such, it seems reasonable to make the wallet receiving fund in a multi-destination tx ignore the encrypted payment ID field and set the associated payment ID to zero regardless.
