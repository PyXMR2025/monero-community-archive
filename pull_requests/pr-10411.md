## selsta | 2026-04-16T15:51:03+00:00
Are these changes here intentional?

## lschomaker1 | 2026-04-16T20:44:08+00:00
Yes. This is intentional and scoped to the Arch CI job only.

  Arch’s current protobuf/abseil packages now require C++17 through the
  optional Trezor dependency path, while release-v0.18 intentionally still
  builds as C++14. This backport does not touch or depend on Trezor, so I
  disabled `USE_DEVICE_TREZOR` for that Arch job to keep CI focused on the
  SOCKS5/IPv6 backport instead of failing on an unrelated rolling-distro
  dependency issue.

  I’m fine splitting or dropping this workflow change if you would prefer to
  keep CI config untouched, but it was needed to get the current Arch job
  green against release-v0.18.
