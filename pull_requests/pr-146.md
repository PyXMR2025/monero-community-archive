# Original Description
## What
This proposal builds upon the previous discussion and sets the initial rough design for Cuprate's RPC libraries, for the purpose of acting as a guide for implementation, although things will change as it is implemented.

This PR only adds the crates in `# Where`.

Closes https://github.com/Cuprate/cuprate/issues/106.

## Where
```bash
# Repo root.
rpc/
тФВ
тФЬтФА json-rpc/
тФВ  тФФтФАтФА [...]
тФВ
тФЬтФА monero-rpc-types/
тФВ  тФФтФАтФА [...]
тФВ
тФЬтФА cuprate-rpc-interface/
тФВ  тФФтФАтФА [...]
```

| Crate | Purpose | Monero (rough) equivalent |
|-------|---------|---------------------------|
| `json-rpc` | JSON-RPC 2.0 library | [`rpc/message.{cpp,h}`](https://github.com/monero-project/monero/blob/cc73fe71162d564ffda8e549b79a350bca53c454/src/rpc/message.cpp) |
| `monero-rpc-types` | (De)serializable Monero RPC type definitions, e.g. `GetBlock`. | [`rpc/daemon_messages.{cpp,h}`](https://github.com/monero-project/monero/blob/cc73fe71162d564ffda8e549b79a350bca53c454/src/rpc/daemon_messages.h)
| `cuprate-rpc-interface` | RPC interface library, uses `monero-rpc-types`, exposes a barebones `axum::Router`, configuration, types, etc. Internally implements routes that handle all pre-handling operations like `restricted`, limits, etc. Defines the `tower::Service` request/response types used by users. Expects users to bring their own handler and middleware. | [`rpc/core_rpc_server*.{cpp,h}`](https://github.com/monero-project/monero/blob/cc73fe71162d564ffda8e549b79a350bca53c454/src/rpc/core_rpc_server.h) |

## How
### `json-rpc`
Rough code:
```rust
// `method/params` -> `body` reasoning:
// <https://github.com/Cuprate/cuprate/pull/146#issuecomment-2148576495>
#[derive(Deserialize, Serialize)]
struct Request<M /* method */> {
    jsonrpc: Version,
    id: Id,
    #[serde(flatten)]
    body: M,
}

/* JSON-RPC 2.0 impl */
```

### `monero-rpc-types`
Rough code:
```rust
#[derive(EpeeAndOrSerde)]
pub struct GetBlock {/*...*/}

#[derive(EpeeAndOrSerde)]
pub struct GetInfo {/*...*/}

/* all RPC types */
```

### `cuprate-rpc-interface`
Rough code:
```rust
// State the RPC router has access to.
pub struct RpcState<S, H> {/*...*/}

// Public function that creates _the_ router.
// Sets all the routes Monero has.
pub fn create_router<S, H>() -> Router<RpcState<S, H>>
where
    H: tower::Service<crate::types::Request>,
    H: <H as Service>::Response: Into<crate::types::Response>,
    H: <H as Service>::Error: Into<crate::types::RpcError>,
{
    Router::new()
        .route("/json_rpc",   get(crate::route::json_rpc))
        .route("/*bin",       get(crate::route::binary))
        .route("/get_height", get(crate::route::get_height))
        /* all routes */
}

// Internal route functions.
mod route {
    async fn json_rpc(/*...*/)   {/*...*/}
    async fn binary(/*...*/)     {/*...*/}
    async fn get_height(/*...*/) {/*...*/}
    /* all routes */
}

// Types derive (de)serialization where needed.
mod types {
    // Could be 1 enum instead.
    pub enum JsonRequestEnum  {/*...*/}
    pub enum BinRequestEnum   {/*...*/}
    pub enum OtherRequestEnum {/*...*/}

    pub enum Request {
        Json(JsonRequestEnum),
        Bin(BinRequestEnum),
	Other(OtherRequestEnum),
    }

    pub enum Response {
        Json(JsonResponseEnum),
        Bin(BinResponseEnum),
        Other(OtherResponseEnum),
    }

    // errors that can occur during RPC.
    pub enum RpcError {/*...*/}
}

// Other stuff.
mod config    {/*...*/}
mod constants {/*...*/}
mod error     {/*...*/}
/*...*/
```
