# Original Description
`t_daemon::run()` declares `stop` and `shutdown` as stack-local atomics (daemon.cpp:238) and captures them by reference into a lambda installed as the global signal handler (daemon.cpp:249). `signal_handler::m_handler` is static and persists after `run()` returns. During the destructor chain (`~t_daemon -> ~t_internals -> ~t_p2p -> deinit -> kill`), thread joins block while the handler still holds dangling references to the deallocated stack frame. A signal arriving in this window writes to dead stack.

Reproduce: ASAN build, start monerod, Ctrl+C twice during shutdown.
```
Build: cmake -D SANITIZE=ON -D CMAKE_BUILD_TYPE=Debug -D STATIC=ON -D BUILD_SHARED_LIBS=OFF
```
