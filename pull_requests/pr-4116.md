However, both implementations have the same issue - the compiler is free to do what it wants, provided the observable effects are the same.  `std::memcpy` has the advantage that its quicker to audit what the code wants to do, and the compiler automatically recognizes the call for optimizations. If you turned down optimizations in this file in particular, then maybe it makes sense, but I would still be critical. Its difficult to control compiler output.
