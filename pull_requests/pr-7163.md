# Original Description
Taking care of possible high resource utilization due to simultaneous local daemon syncing and wallet refresh is out of the Wallet API scope.
Should be handled on the client side instead, e.g. checking `busy_syncing` daemon flag and invoking`pauseRefresh()`/`startRefresh()` Wallet APIs.
