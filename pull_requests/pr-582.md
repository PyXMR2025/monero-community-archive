# Original Description
### What
Closes #542 by making Arti an opt-in feature

### Why
Reduce default required dependencies

### Where
cuprated, p2p

### How
Grouped Arti-exclusive imports under a single feature-flagged `use` block, feature flagged parts of Arti related code that rely on said imports, marked Arti crates as optional.

for `TorMode` resolution, if Cuprate was compiled without Arti but `Auto` is set, make it immediately fall back to Daemon as its the only valid option left.

With this PR, Arti is not in the default set of features anymore (`default=["static"]` was removed), has to be explicitly enabled with `--features arti`.
