## hinto-janai | 2024-07-12T23:54:16+00:00
> I would prefer using a crate, unless there is a good reason for using a homegrown one?

1. IIRC there's no stack string crate that everyone uses, the closest I think would be [heapless](https://github.com/rust-embedded/heapless)
1. We'd be exposing a 3rd party crate in our API, not the worst but a bit awkward IMO
2. `epee_encoding` would stop working, we'd have to make a wrapper for impl

We could also enable features to swap certain types, e.g.:
```rust
cfg_if! {
    if #[cfg(feature = "heapless")] else {
        pub type String = heapless::String;
    } else {
        pub type String = String
    }
}
```
We could do this for `ByteArray` stuff too, i.e. just keep all types as lenient as possible but allow type optimizations via features.
