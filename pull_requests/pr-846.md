# Original Description
The compiler treated some warnings as errors, so fixed those.

The compiler complained about exceptions thrown from destructors. The commits in this PR add `noexcept(false)` specs to these destructors, but perhaps another fix would be to avoid throws (most or all of which seem to be non-essential logging/error-handling code).

NOTE: I regenerated the makefile with cmake so some compiler flag came from system (mostly default makepkg.conf). Btw, might be good to remove generated Makefiles from VCS.
