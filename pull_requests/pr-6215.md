# Original Description
RPC clients ban feature counts how many errors every host has made and after it gets more than certain threshold the host is banned for 24 hours.  

Often public monero nodes are not exposed directly to internet but resides behind web server or a proxy. In this valid case the only remote host the node sees is proxy server's one. If one of external clients makes some RPC calls that end up with ban the RPC becomes unavailable for all the clients.

This is what happened with our public node lately.

If `--disable-rpc-ban` option is set the feature is disabled.
