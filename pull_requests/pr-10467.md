# Original Description
`DaemonHandler::handle(GetBlockHeadersByHeight)` in `src/rpc/daemon_handler.cpp` did not cap `req.heights` under `--restricted-rpc`, so a restricted caller could request an unbounded number of block headers in a single ZMQ call.

Add an early return when `req.heights.size() > restricted_max_block_headers`, using the same 1000-request limit as HTTP `on_get_blocks_by_height` in `core_rpc_server.cpp`.

Follow-up to #10424.
