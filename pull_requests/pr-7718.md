## selsta | 2021-07-13T05:48:48+00:00
> (if I understand it correctly)

This just adds `-Werror` to the check_c_compiler_flag test. Replacing `-` with `_` is already in the current code.

An example would be here: https://github.com/monero-project/monero/blob/master/CMakeLists.txt#L773-L774

Clang produces the following warning but no error:

```
clang: warning: argument unused during compilation: '-fstack-clash-protection' [-Wunused-command-line-argument]
```

We don't want to add any flags that produce warnings so we add `-Werror`. With this patch the flag does not get added and no warning shows up during compilation, as expected.
