Btw the rough reasoning behind why this [lint](https://rust-lang.github.io/rust-clippy/master/index.html#missing_const_for_fn) was added:
- `const fn` are a subset of `fn`; the signature gives more semantic information about what the function is allowed to do e.g. no allocation
- If something can be more restricted, why not

The exception/con is with public APIs, using `const` then removing it later would be a breaking API change; adding `const` should be given more thought in these situations. The majority of our `const fn` are internal so I think this is fine for now but it will pop up when opening our crates publicly with a stable API.
