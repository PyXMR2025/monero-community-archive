This should (hopefully) improve UX a tiny bit.

Is there a reason why it can't be realized via:
`while (loop_counter++ <= 3)` ?

If there is, then it's safer* to write:
`if (loop_counter++ > 3)`
and remove line 6536 (the incrementation).

*safer, bc you then have a guarantee, that the incrementation occurs whatever happens within the loop, also regarding somebody else's future changes.

Ideally, the `3` should be declared as a constant, so finally:

```C++
const int MAX_UI_LOOPS = 3;
int loop_counter = 0;
bool success = false;
while (loop_counter++ <= MAX_UI_LOOPS && !success)
{
 /*... */
    try
    {
        if (get_and_verify_password())
        {
           success = true;
           // break; is optional, as the loop will be controlled by the boolean, as it's supposed to.
       }
    }
    catch()
    /*... */
}

if (! success)
{
   // Log error
  // Throw exception
}

```

or:

```C++
const int MAX_UI_LOOPS = 3;
int loop_counter = 0;
while (1)
{
/* ...  */
    try
    {
        if (get_and_verify_password())
        break;
    }
    if (loop_counter++ > MAX_UI_LOOPS)
    {
        tools::msg_writer() << inactivity_msg << (inactivity_msg[0] ? " " : "") << tr("Failed to provide the right password after ") << MAX_UI_LOOPS << tr(" tries".);
        // throw some exception instead of just exiting, to clean up memory and close streams.
    }
}
```

Correct me if I'm wrong on the details above (like how to log and what kind of exception to throw), but this would be my entry point.
