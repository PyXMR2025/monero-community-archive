## moneromooo-monero | 2018-10-12T13:45:31+00:00
Why did you take out the DEPENDS check ?

## xiphon | 2018-10-12T14:22:53+00:00
This check is the main reason we have 0.13 builds broken on armv7.
We don't need it. I think this kind of thing should be done on the Travis or Depends side.

This check is only needed for Travis+Depends darwin build, but take a look at what it actually does:
1. Depends is invoked before the Monero's CMake
2. Depends sets ARCH='x86_64'
3. Monero's CMake invoked
4. Due to this check, Monero's CMake reverts  ARCH to ARCH='native'
5. We compile so-called darwin x86_64 build with arch 'native'. This doesn't make any sense. Is not actually a cross-compilation.
