# Original Description
Securely erase keys from memory when they go out of scope. Might have a
performance impact, which I haven't measured (perf measurements aren't
generally reliable on laptops).

I wasn't entirely happy with having to add the `scrubbed::inner()` methods, but some `static_assert`s in the serialization stuff complains that the wrapper class's dtor causes there to be padding, for reasons I don't yet understand. I considered hiding it slightly with an `operator*()`, but wasn't sure what the norms were in this project toward that. I'm happy to take advice/feedback here.
