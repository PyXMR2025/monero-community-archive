# Original Description

### What

Adds the DB requests and handlers for blockchain history, this history is sent to peers so they can send us their chain to sync from.

### How

Uses the same technique as monerod, choosing the first 11 blocks from the top of the chain and then power of 2 offsets from there.

## hinto-janai | 2024-07-02T21:24:11+00:00
I'm not sure this is correct? Sorting is not enforced or documented either.

AFAICT this binary search requires the input slice to be partitioned like so:
```
[exists, exists, ..., doesnt_exist, doesnt_exist]
```
but:
1. The input `block_ids: &[BlockHash]` isn't enforced to be sorted like this
2. There is no sort order for "exists", if there were, the caller would know the index already

For example:
```rust
fn main() {
    let x = [0; 32]; // known hash
    let y = [1; 32]; // unknown hash

    let block_ids = [x, y, x, x];
    let blockchain = [x];

    // should return 1
    let idx = block_ids.partition_point(|id| blockchain.contains(id));
    assert_eq!(idx, 4); // hmm
}
```
