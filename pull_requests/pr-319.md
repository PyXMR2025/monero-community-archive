## jeffro256 | 2026-04-20T19:25:29+00:00
`n_proofs - i` can be improved to `std::min(n_proofs - i, max_batch_size)`, where `max_batch_size = n_proofs / n_batches + max_bin_ratio`, where `max_bin_ratio = div_ceil(max_n_inputs_in_proof, min_n_inputs_in_proof)`. `max_bin_ratio` need only be calculated once at the beginning of the function, after the pointer list is sorted.

This is because the maximum difference b/t the sums of bins cannot be greater than the maximum item size. And if the size of the minimum element > 1, then you can divide this maximum difference by fewer elements. Since the maximum FCMP++ input count is 128, then the maximum over-allocation of elements is $128 * k$. Whereas, without the change it is $(n - 1) * k$. 

This should really help keep the memory usage down if `batchVerifyFcmpPpProofs()` is called with many proofs.
