# Original Description
Without this commit:
1) read height from DB
2) add block to chain in separate thread
3) read chain for block id's and request them from peer
4) ERR in `handle_response_chain_entry`, peer's first block is the one that was added to the chain, which has block idx=height from step 1.

This commit reads the chain for height and highest block id's in one go while holding the `m_blockchain_lock` to avoid the race.
