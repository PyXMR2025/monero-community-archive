# Original Description
This is yet another alternative to #1943 with the difference being that it requires signer's spend secret key to really prove that the signature could have been only created by the wallet owner. The modified signature scheme now proves that the signer knows secret keys `x` and `y` such that `P=x*G` and `Q=y*A`.

Pseudocode:

```
[Signature Generation]
Input: Msg, x, y, A (where P=x*G and Q=y*A)
    k = random_scalar()
    R1 = k*G
    l = random_scalar()
    R2 = l*A
    c1 = Hs(Msg || R1 || R2)
    r1 = k - c1*x
    c2 = random_scalar()
    r2 = l - c2*y
    return (c1, r1, c2, r2)

[Signature Checking]
Input: Msg, P, Q, A, c1, r1, c2, r2
    R1 = c1*P + r1*G
    R2 = c2*Q + r2*A
    c1' = Hs(Msg || R1 || R2)
    return c1 == c1'
```

(Implemented as `generate_signature_2` & `check_signature_2` in `src/crypto/crypto.cpp`)

The command syntax is slightly changed:

```
get_tx_proof <txid> <dest_address> [<tx_key>]
check_tx_proof <txid> <dest_address> <signer_address> <signature>
```
