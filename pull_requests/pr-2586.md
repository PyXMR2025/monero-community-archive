# Original Description
This functionality is similar to the `do_not_relay` option in the wallet RPC, which I think would be useful for pushing tx anonymously to a remote node on Tor or I2P.

Like the wallet RPC, I could have made it as a per-transfer option by changing the command syntax of  `transfer` (e.g. introduce additional keyword `do_not_relay` at the end), but I thought that'd be cumbersome.
