## vtnerd | 2020-10-20T16:28:38+00:00
With C++11 `constexpr` this is just:
```c++
constexpr std::uint64_t powerof(std::uint64_t a, std::uint64_t b) noexcept
{
    return b == 0 ? 1 : a * powerof(a, b - 1);
}
```
which is computed at compile time with `-O1`, `-02` or `-Ofast` (basically any optimizations enabled). A `constexpr uint64_t`  could be placed in the cpp file which forces compile time calculation even with optimizations disabled. No static memory is used if the address is ignored. This works with at least Gcc 4.9.4 (maybe further back).

## vtnerd | 2020-10-21T13:48:37+00:00
`constexpr` is header-only ? And I doubt there is a difference in practice after compiler optimizations. The primary difference is the use of an `enum` in the inner class (presumably to prevent storage for an inner constant), which is still valid with `constexpr`:

```c++
constexpr std::uint64_t powerof(std::uint64_t a, std::uint64_t b) noexcept
{
  return b == 0 ? 1 : a * powerof(a, b - 1);
}

enum constants : uint64_t
{
  quantization_mask = powerof(10, CRYPTONOTE_DISPLAY_DECIMAL_POINT - PER_KB_FEE_QUANTIZATION_DECIMALS)
};
```
