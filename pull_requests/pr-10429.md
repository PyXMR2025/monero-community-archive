# Original Description
Defer loading of private view key from device until first time it is needed. Do not fail if this fails. This has two effects:

1. The prompt to export the view key is only needed once when creating a `cryptonote::account_base` (it shows twice currently)
2. The call to `connect()` doesn't fail if the user decides to not export the viewkey, and thus usage of the device without exporting view keys is possible

This is a small convenience for Ledger users, but will be an even larger convenience for Ledger device testing
