# Original Description
### What
  Closes #486

  Implements hourly logging to detect when no incoming P2P connections are established.

  - Adds hourly monitoring that logs a warning when no inbound connections are active
  - Refactors semaphore creation for shared access between monitoring and connection handling
  - Skips monitoring when inbound connections are disabled

  ### Why

  Users may not realize their node isn't receiving incoming connections due to firewall/NAT issues.

  ### Where

  - P2P networking layer
  - Background task management
  - Function signatures for `inbound_server`

  ### How

  1. Moves semaphore creation to `initialize_network` for shared access
  2. Spawns monitoring task that checks semaphore permits hourly
  3. Logs warning when all permits are available (no active connections)
