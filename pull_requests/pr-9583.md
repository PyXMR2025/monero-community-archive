# Original Description
There's _technically_ an overflow in `byte_slice.cpp` that I caught while simply scanning the code. This can occur if the total memory being copied exceeds `size_t`. This is unlikely to ever trigger, because the platform probably needs `sizeof(size_t) == 4 && sizeof(void*) == 8` (or similar) to occur. And somehow a `malloc` to copy all of this data needs to succeed too. Therefore, this is primarily just defensive programming to be strictly standards compliant.
