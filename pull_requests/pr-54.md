# Original Description
Before we were batching blocks together and getting outputs for all transactions in the batch in 1 DB request, this significantly sped up scanning time.

However a tx could reference an output in the batch, so it will not be in the DB, this was not an issue when using a fully synced node as a database as they already have all outputs, but when syncing this will cause issues.

To fix this I have added a new `OutputCache` that will cache all outputs in the batch so we can get outs from this and the DB (I don't think cache is a good word but I can't think of a better one)

I need to update the `scan_chain` binary.
