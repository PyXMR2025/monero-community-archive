# Original Description
This implements the idea proposed in Issue #2065: for each input ring with pubkeys `(P_1, ..., P_N)` and a key image `I` in a given transaction, each part of the signature proves that the signer knows the secret key `x` such that `P_s = x*G` and `I = x*Hp(P_s)` for a secret index `s`. I.e., this is the most basic CN ring signature (without any other advanced stuff like CT).

Note that this method require the tx secret key, hence doesn't prove the sent amount to any recipient addresses; it only proves that the signer must have created the given tx.

This method doesn't seem to introduce any harm privacy-wise, in contrast to my recent PR #2366.
