# Original Description
An update by @jeffro256 changed how the txpool set `m_added_to_pool` - it only set the value on first observation as opposed to first fluff/block observation. Unfortunately, the ZMQ-PUB code was designed to only report txes observed during block/fluff (suppressing "knowledge" of the tx during stem phase).

This means a node taking part in the stem phase of a tx never reports the tx over ZMQ currently. This reverts back to the prior behavior by using the return code from `txpool::add_tx`.

Watching the CI, and then pushing a 0.18 patch too.
