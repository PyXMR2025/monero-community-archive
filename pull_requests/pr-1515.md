## pazos | 2018-07-19T23:07:28+00:00
AFAIK there is no need to do dynamic memory allocation here. 

`QSettings settings(QSettings::NativeFormat, QSettings::UserScope, app.organizationName(), app.applicationName());`

should work.

Also please take QSettings::IniFormat in consideration, because it is less invasive on windows and portable across OSes.

## sanderfoobar | 2018-07-21T13:16:40+00:00
@pazos We can't switch to `QSettings::IniFormat` without some sort of migration though.

On Windows, registry->.ini
On OSX, plist->.ini (?)
On Linux, .conf->.ini

## pazos | 2018-07-23T23:17:21+00:00
linux stores settings in ini, it doesn't have its own native format.

migration is ok, but can lead to bugs and add ugly code. IMO is better to do the switch in a major release and never look back 😄
