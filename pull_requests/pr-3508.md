# Original Description
Closes #3507, #2328

Still work in progress. Tested only on Trezor.

Changes to hardware wallet wizard page (WizardCreateDevice1.qml): 
- Explanatory title and subtitle
- Wallet location displayed only on advanced mode (https://github.com/monero-project/monero-gui/issues/2328#issuecomment-557084158)
- Differentiate Ledger models (Nano S x Nano X), as they have different firmwares and different minimum restore heights (see below)
- Add device photos
- Add minimum required firmware versions
- Add explanation to restore ("My Trezor/Ledger already holds Monero") (#2328)
- Add prefilled minimum restore heights values for each device model, according to when they started supporting Monero (see https://github.com/monero-project/monero/issues/6797)
- Subaddress lookahead displayed only on advanced mode

New HardwareWalletDialog component:
- Displayed when Monero GUI can't connect to hardware wallet (when opening a wallet file or when creating a new wallet file from device)
- Device photo
- Instructions on how to solve most common issues (troubleshooting)

## rating89us | 2021-05-27T18:44:32+00:00
The idea here is to display a nice message every time the user opens a wallet file of a hardware wallet, but trezor/ledger device is locked. 

I usually forget to unlock my trezor before clicking on the wallet in "open wallet" page, and then I get this error message "Could not connect to the device Trezor" that doesn't inform that the wallet is locked.

Ideally we should know before opening the wallet file that it is a hardware wallet file, and then we display this nice reminder that the user should unlock the hardware wallet first. However, it is not possible to see if the wallet file is from a hardware wallet before opening and decrypting it. An alternative would be to save the wallet files from hardware wallets on a special directory, but this has privacy implications.

Anyway the code works this way:
Open wallet file -> can't connect to hardware wallet -> display nice message to remind user to unlock it (on first try) -> `firstHardwareWalletConnection ` set to `false ` -> can't connect again -> display troubleshooting message
