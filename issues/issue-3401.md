## snipeTR | 2025-06-16T21:04:23+00:00
> [@geekwilliams](https://github.com/geekwilliams): Because XMRig is not a person but a project.

It’s important to clarify that XMRig is both the name of the project and the pseudonymous identity of the individual developer maintaining it.

The current GitHub account (https://github.com/xmrig) functions as a personal account, but it was intentionally created and maintained under the project’s name. This is not unusual in the open-source world — many developers use the project name as their GitHub username, especially in solo-maintained or community-driven projects.

So while @Neustradamus suggests that the project should be converted into an organization ([GitHub doc](https://help.github.com/articles/converting-a-user-into-an-organization/)), this assumes a clear separation between person and project, which does not apply in this case. The current setup complies with GitHub’s [Account Requirements](https://docs.github.com/en/site-policy/github-terms/github-terms-of-service#3-account-requirements), as the account clearly belongs to a human being who meets those terms.

Additionally, while it's technically possible to rename the user account ([docs](https://docs.github.com/account-and-profile/setting-up-and-managing-your-personal-account-on-github/managing-user-account-settings/changing-your-github-username)), create an organization ([docs](https://help.github.com/en/github/setting-up-and-managing-organizations-and-teams/creating-a-new-organization-from-scratch)), and then transfer repositories ([docs](https://help.github.com/en/github/administering-a-repository/transferring-a-repository)), these are just options—not obligations. Every project maintainer has the right to choose the structure that suits them best, especially when the project and the developer share a common identity.

In short, XMRig is not impersonating a project; it is the project, and it is the developer. The current setup reflects that dual nature.

## Neustradamus | 2025-06-16T22:01:17+00:00
@xmrig: There are two possibilities (several links are in previous messages):
- Converting this personal account and select the good OWNER (your real personal account)
OR
- Rename your account, create the good org (free), and transfer repositories into the new org

If you do not want to convert this personal account (and select the good organization personal account owner):
- You can rename this personal account to another username (it takes few seconds here: https://github.com/settings/admin)
- Create the good organization following the first step (it takes few seconds here with the good username: https://github.com/organizations/plan)
- Transfer all xmrig repositories from the personal account into the organization (it takes few seconds)

ALL redirections will be here, no lost.

Several users and projects have done it without any problem.
