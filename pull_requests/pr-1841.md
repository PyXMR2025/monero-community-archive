## vtnerd | 2017-03-04T22:09:03+00:00
 The too-easy-to-use `[&]` is definitely deceptive with asynchronous calls. If the shared_ptr is copied when creating the lambda => `[control] () { download_thread(control); }`, the code can immediately return. `boost::thread` will heap-allocate a copy of the lambda, so the reference count remains active until the function on the other thread returns. Alternatively, `control->thread = boost::thread(&download_thread, control);` always copies its arguments if that's any easier.
