# Original Description
Currently, the config file is read after some other args have been read, which means that if those settings are specified in the config file, they aren't reflected in the program. Here's one particular problem reported in #3603: if one launches the daemon as

    monerod --config-file /etc/test_monerod.conf

with the config file containing:

    testnet=1

the daemon log gets written to the  *mainnet* log file (`~/.bitmonero/bitmonero.log`) instead of the testnet log file (`~/.bitmonero/testnet/bitmonero.log`) because the daemon log path is determined before reading the config file.

Also, I've removed the following bit

```c++
    if (!config_path.has_parent_path())
    {
      config_path = data_dir / config_path;
    }
```

which I found confusing, because the default data dir depends on the network type (mainnet/testnet/stagenet) and the network type could be specified in the config file (also, the data dir arg could be specified in the config file). A much simpler way seems to be to always specify the full path of the config file. I hope there aren't many users affected by this change.
