# Original Description
Currently we write the Monero GUI configuration to:

- The registry on Windows
- XML preferences file on MacOS
- `~/.config/monero-project/monero-core.conf` on Linux

We use `import Qt.labs.settings 1.0` to handle settings for us (called `persistentSettings`). This QML component allows  to specify an arbitrary filepath, except, that this is [not available for our Qt version](https://doc.qt.io/qt-5/qml-qt-labs-settings-settings.html#fileName-prop). This means we cannot write to a location at will.

This PR is a backport of the `Qt.labs.settings` [QML plugin from 5.12.0](https://github.com/qt/qtdeclarative/blob/5.12/src/imports/settings), aliased as `MoneroSettings`. 

The nice thing is that we don't have to change anything in our QML except for:

```qml
Settings {    <== Change this to `MoneroSettings`
    id: persistentSettings
    property bool foo: true
    [...]
}
```

Writing to a specific path:

```qml
MoneroSettings {
    id: persistentSettings
    fileName: "/tmp/foo.conf"
    property bool foo: true
}
```

This comes in handy for:
- Tails support (#2258) 
- Portable installations
