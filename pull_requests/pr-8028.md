# Original Description
This is an update to the documentation on the levin protocol to describe my proposal for end-to-end encryption. The proposal recommends `Noise_IK`, but I definitely encourage criticism on whether TLS 1.3 should be used instead. I am not certain of the choice myself due to the maintenance burden this placed on the project (its no worse than the custom crypto for ringct, at least).

The primary advantage of the Noise protocol is the increased difficulty in identifying the traffic. As mentioned in the document, identification is still likely possible, so the Noise protocol is primarily just increasing the effort required. The past issues with OpenSSL (and SSL/TLS protocol) have also led some people to prefer the Noise protocol, but this is hard to quantify.
