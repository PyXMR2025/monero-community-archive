# Original Description
There was a subtle bug in the histogram calculation in tx_memory_pool::get_transaction_stats that could, under uncommon conditions, result in an out-of-bounds write. For more information, see the extended description of the commit, which fixes this bug.

This bug was found while experimenting with speeding up adding transactions to the mempool using parallelisation.
