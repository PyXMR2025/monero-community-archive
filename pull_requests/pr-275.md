# Original Description
This prevents a data race where the `request_missing_objects` function ends up thinking a block span is missing when really the block(s) just got added to the db.

This seems to be the last remaining piece to solve runaway span puzzle.

A stressnet user who was consistently reporting OOM's due to the runaway spans applied this PR, and it stopped the OOM's.

I wish there was a cleaner solution to this race, but I don't see one that doesn't involve completely rewriting this protocol.
