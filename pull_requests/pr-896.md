# Original Description
Shorten the list of warnings that are reported, but
which are forced to NOT generate an error, via -Wno-error.

Unwhitelist these: strict-aliasing, sign-compare, type-limits

For example, ignoring strict-aliasing warning caused
lots of wasted time diagnosing Issue #847.

---

Tested build on: x86 64-bit (gcc 4.8) and 32-bit (gcc 6.1), and on ARM 32-bit (gcc 6.1). On the latter two the aliasing warning is produced, and on the ARM 32-bit the built binary is broken (#847, `blockchain_import --verify 1` fails on block 4), on x86 32-bit, the import does not fail on block 4, so perhaps the binary is ok despite the warning. Even so, it's still better to remove the exception and let that warning be fatal on x86 32-bit as well, if we are letting all warnings be fatal (`-Werror`) to begin with.

**Please merge after #887 and #886**, to avoid needlessly breaking builds on platforms where the aliasing warning is issued, but the build binary does not come out broken (x86 32-bit); and to have the tests build despite these warnings.
