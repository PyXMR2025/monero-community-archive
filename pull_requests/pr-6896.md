# Original Description
I have seen a case where the current locking is not enough to prevent
corruption when calculating hashes from both the current height seed
hash and other seed hashes. This happened in townforge, when syncing
historical merge mined blocks (hence the mix of current and arbitrary
seed hashes) but it seems to also apply to monero since one can ask
for pow calculations on arbitrary blocks while syncing, though I have
not actually tried to repro this on monero.

Thanks to hyc for discussion and help debugging.
