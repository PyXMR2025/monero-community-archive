# Original Description
In C++11, an overridden virtual method may be decorated with the `override` keyword, which delivers a compile-time check whether the code works as intended or not - that the method, that is expected to work polymorphic, will indeed work so. If, after such decoration, the code fails to compile, then either the base class' method wasn't marked as virtual, or there was a typo in either the base class or the overridden class. The `override` keyword binds these methods for good.

Ignoring this C++11 feature causes compilation warnings under CLang (although not in our CI for some reason).
