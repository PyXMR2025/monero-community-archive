# Original Description
Hi I recently tried to start and control xmrig from another process but this didn't work, as you connect directly to the TTY.

I added a fallback option to a regular pipe in case the TTY is not available so I can send the `h, c, p, r, s` commands from another process, Of course due to the nature of stdin this only works if you flush the output to the stdin pipe every time you write a character, so I added a comment explaining that in the code as well.
