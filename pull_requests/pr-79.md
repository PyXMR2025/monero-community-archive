# Original Description
If the wallet is attempting to process a set of blocks it has already processed, we can short-circuit early in `process_parsed_blocks`.

The wallet might request already processed blocks [when seeking to protect against a reorg](https://github.com/monero-project/monero/blob/fbc242d52d89d9c8021194cd4faae657c94d5a31/src/wallet/wallet2.cpp#L3478-L3484).

This PR should fix a scanning bug reported by @nahuhh where the wallet was requesting already scanned blocks and then throwing in `tree_sync_blocks_async` (`tree_sync_blocks_async` throws when attempting to process a complete set of blocks it has already processed).

I have a larger PR on the way that removes the init block hash downloading, makes it even clearer when the wallet requests already scanned blocks, and largely simplifies wallet2's refresh imo.
