# Original Description
it would be nice if you could set multiple access for wallets within the Monero_Wallet-GUI api.

allow for using all three net-types (mainnet, testnet, stagenet), potentially increasing productivity of the Api. by allowing send/receive between wallets as well as possibly allowing for possible running of multiple mining of pools as well as the default pools....
this would potentially allow for multiple sessions of monerod.exe, altho syncing between wallets would have be dynamic on the computer so that updating could be handled in a few instances instead of of just one potentially creating faster updating of blockchain data file.

this could potentially be added to XMRig mining Api. as well... would also confuse the hell out of any passive data collection that might be used by the ISPs/etc..

last suggestion here:
make it so that the Api. will automatically know (based on the wallet info.) what net that wallet is attached to so you do not have to keep going back to change the net connection...
(this should go along with remembering language options for each wallet as well)
