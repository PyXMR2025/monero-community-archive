The call flow goes like 

- `thread-wallet-api`: `onDevicePassphraseRequest` -> `onWalletPassphraseNeeded` mutex & cond wait for UI, wait
- `thread-gui`: UI calls `onPassphraseEntered`, fills in data, wakes condition variable which causes unblocking of the `thread-wallet-api`
