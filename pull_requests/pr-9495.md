# Original Description
Building on #10201, this change completes the mechanism to contain runaway spans. It modifies `queue_proceed` and replaces the previous static `span_limit` with a dynamically calculated limit.

The dynamic span limit estimates how many spans can be synced within N minutes (default: 2) and is configurable via `--span-limit=<minutes>`. A soft 100 MB cap on block-download size is applied (adjustable with the existing `--block-download-max-size=<bytes>`). Both the span and size limits may be temporarily exceeded during short bursts or when the next span to be synced is unavailable; otherwise, they are enforced to keep sync activity contained.

Stressnet version: https://github.com/seraphis-migration/monero/pull/234
