# Original Description
The tests `wallet-crypto-bench` and `cncrypto` crash on Linux running on sparc64 due to unaligned access (Bus error):

```
The following tests FAILED:
	  2 - wallet-crypto-bench (Bus error)
	  3 - cncrypto (Bus error)
Errors while running CTest
```

Full build log available in: https://buildd.debian.org/status/fetch.php?pkg=monero&arch=sparc64&ver=0.18.4.2%2B%7E0%2B20200826-1&stamp=1757525344&raw=0

While sparc64 is not a mainstream platform, fixing the cause for the unaligned accesses on sparc64 might help fix the issues reported in #9255, #8889, #8837 and #2669.

Access to SPARC machines for testing can be obtained through the GCC Compile Farm: https://gcc.gnu.org/wiki/CompileFarm
