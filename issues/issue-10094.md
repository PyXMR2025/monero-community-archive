## ndorf | 2025-10-01T00:58:11+00:00
Did you build your own monerod from source, and if so, did you do it on a different machine than the one it's running on?

By default, the build targets the specific CPU you build on. If you want a portable binary, you have to pass the `-DARCH=default` argument to cmake. Otherwise, the compiler can output newer instructions, unsupported by older CPUs, and you'd see exactly this error.

(You can also target the CPU you want to run on, e.g. `-DARCH=sandybridge`, but in this case you also have to make sure to pass `-DARCH_ID=x86_64`, otherwise randomx will be built incorrectly and linking against it will fail.)
