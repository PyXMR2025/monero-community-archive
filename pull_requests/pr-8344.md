# Original Description
## Motivation
This PR adds a new file to the data directory: `rpc_ssl.fingerprint`. This is a convenience file that stores the SHA-256 X509 digest of the PEM certificate at `rpc_ssl.crt`. This is ease-of-use improvement for those who want to manually add their node's fingerprint to the CLI wallet / other application (and hopefully GUI eventually, working on that currently). Now there aren't any commands to memorize; no searching for which types of fingerprints `net_ssl` uses. I hope this encourages more users to utilize secure connections over RPC.

## PR structure

There are 4 four part of this PR:
1. Move `get_fingerprint` function from `gen_multisig` to `net_ssl` (to be used below)
2. In `store_ssl_keys` (when storing the `.crt` and `.key` files for the first time), we now also store the `.fingerprint` file
3. Add `get_hr_ssl_fingerprint_from_file` which is basically just the command `openssl x509 -in <filename> -fingerprint -sha256 -noout` as a C++ function
4. Finally, for those existing nodes, in `core_rpc_server`, we check if we already have those `.key` and `.crt` files but *not* the `.fingerprint` file. If so, create it.
