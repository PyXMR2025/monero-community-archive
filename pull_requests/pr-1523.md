# Original Description
This adds HTTP client "authorization" field generation to the same library that contains the HTTP server authentication. This does not add the "hooks" to the epee code to generate the authorization field. Client authentication will be necessary in the wallets if HTTP auth is added to the `monerod` RPC daemon.

Client authorization uses the exact same parsing framework that the server code uses. I apologize for the renaming spam - I did not plan _completely_ ahead of time.

This supercedes [PR 1514](https://github.com/monero-project/monero/pull/1514), which I will leave up because I expect this will have more discussion.
