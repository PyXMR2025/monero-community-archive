# Original Description
It's 2021, we can easily switch to C++14 without any troubles.

I also looked into C++17 but it would require us to drop macOS 10.10 - 10.13 support, which wouldn't be ideal, so C++14 for now.

This doesn't mean that we should rewrite current code with C++14 features, just that we can (optionally) use new features in new code. The only exception would be if it allows us to drop boost in some places.
