# Original Description
I often find myself executing the same command multiple times in a row, when using monero-wallet-cli. For example, I often call `refresh` until I can see that my balance is confirmed. When I then try to get to a previous command I entered, by pressing the "up" key, I found that I had to press it many times, before I surpassed the `refresh`s.

With this change, the history will not contain consecutive, identical entries anymore. This works both when the same command is re-entered manually and when it's entered by pressing "up" and "return". This is the same behaviour, that I know from [zsh](https://www.zsh.org/).
