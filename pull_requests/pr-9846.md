# Original Description
I found a place where `std::pair<transaction, blobdata>` was given to a function that took a `const std::pair<transaction, blobdata_ref>&` argument. As per C++ rules, this forces a hidden implicit copy of `transaction`, which is a rather expensive operation in the `add_block` routine. Additional, the `miner_tx` was being copied too (and also in a migration function, which is less important).

I haven't done any timing/performance analysis, but this should help block syncing by a tiny amount. If this passes checks, I'm going to propose this for 0.18, except using `blobdata_ref` instead of `span<...>`. I'm using `span<...>` in this patch because I hope to change things over from `blobdata` (aka `std::string`) to `epee::byte_slice`.
