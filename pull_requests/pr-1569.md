I'm not too familiar with these time-related functions, and the cppreference's [sample code](http://en.cppreference.com/w/cpp/chrono/c/mktime) shows:

    std::time_t t = std::time(NULL);
    std::tm tm = *std::localtime(&t);
    std::cout << "Today is           " << std::put_time(&tm, "%c %Z")
              << " and DST is " << (tm.tm_isdst ? "in effect" : "not in effect") << '\n';
    tm.tm_mon -= 100;  // tm_mon is now outside its normal range
    std::mktime(&tm);  // tm_dst is not set to -1; today's DST status is used
    std::cout << "100 months ago was " << std::put_time(&tm, "%c %Z")
              << " and DST was " << (tm.tm_isdst ? "in effect" : "not in effect") << '\n';

So, out-of-range values specified to any fields of `tm` will not cause `mktime` to produce error, instead it'll normalize the values. In other words, it won't reject 2012-04-31, but instead interprets it as 2012-05-01.
