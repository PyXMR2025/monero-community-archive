Added some fixes for building the GUI on windows (through msys2-mingw32), since currently that doesn't work as the guide prescribes. I've added the fixes in a way, which shouldn't mess with build environments already customized and the build-bot.  Also changed the windows build guide in respect to the changes.

Because https://github.com/monero-project/monero-core/blob/master/monero-wallet-gui.pro#L145 specifies a win32 build.

The code you are quoting checks if the host in win32 or win64, as to make sure the msys path is correct (either c:/msys64 or c:/msys32)
