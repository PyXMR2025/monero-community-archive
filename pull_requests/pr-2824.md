# Original Description
Implemented `--verify-update` command-line option.

The code will be used in Monero GUI one-click update downloading and verification mechanism.

Verifies update binary using `shasum`-compatible (SHA256 algo) output signed by two maintainers.  
* Requires `hashes.txt` - signed `shasum` output generated by a maintainer:
  `gpg -o hashes.txt --clear-sign <shasum_output>`
* Requires `hashes.txt.sig` - detached signature of `hashes.txt` generated by another maintainer, signing the whole `hashes.txt` contents (including the first maintainer's signature):
  `gpg -b hashes.txt`

Hardcoded @binaryFate, @fluffypony , @luigi1111  public keys.
Update verification requires using any 2 of them.
```
monero/utils/gpg_keys/binaryfate.asc
monero/utils/gpg_keys/fluffypony.asc
monero/utils/gpg_keys/luigi1111.asc
```

Sample output:
```
$ ./monero-gui --verify-update /some/folder/monero-gui-win-x64-v0.15.0.4.zip
successfully verified, signed by "binaryFate <binaryfate@getmonero.org>" and "luigi1111 <luigi1111w@gmail.com>"
```
