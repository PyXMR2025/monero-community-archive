Because I don't want to go through the effort to get fuzzing to build in windows CI, I have made it by default the fuzz crate is not built.

There are 2 ways to do this, the first is to use `--exclude cuprate-fuzz` whenever you want to run anything with cargothe second is to have this `default-members` in cargo.toml. I went with the second as I feel needing to type --exclude all the time is annoying. You can still build everything (including `cuprate-fuzz`) by using `--workspace`
