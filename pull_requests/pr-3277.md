# Original Description
As per #3017, the current implementation is somewhere between the two opinions:

@fluffypony 
> - Devnet, which is the current testnet renamed. It lets us test future forks and features live.
> - Testnet, which is a new net, new ports, and matches the live environment by forking at roughly the same time.
> - Mainnet, stays as-is.

@hyc 
> I'm kinda inclined to keep testnet with its current network and ports, and come up with a new name for the newly created staging network. 

I.e. the testnet is the same as before, and a new network called 'devnet' is introduced. If we agree to swap these names, I can easily do that.

Also, further simplification of the command line flags will be possible once #3170 gets in.

The network is up and running at a very low hashrate with two seed nodes at 162.210.173.150:38080 and 162.210.173.151:38080.
