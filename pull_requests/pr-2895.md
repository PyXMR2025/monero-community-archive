Should we start (covertly) tagging these in some way? An example would be appending a MAC (poly1305?) where the input was `MAC(key, "MONERO-WALLET--FILE-V2" || encrypted data);` The wallet would still just be encrypted bytes (with an implied MAC), so it would contain no identifiable information. And new wallet encryption schemes can be detected easily since the probability of the MAC succeeding is low.

In the current implementation if the MAC failed, it would just jump to the older chacha8 algorithm (can't reject it, unfortunately).
