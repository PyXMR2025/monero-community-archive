# Original Description
1. Drop the code dealing with executable file's sections like `getsectiondata`, etc.
2. Replace custom `LD -r -b binary` command with auto-generated `.c` files
3. Drop `libblocks` dependency from `cryptonote_core` and `device`. 
4. Drop `blocks.dat` empty file that was used to build `cryptonote_core` and `device`.

Checkpoints data files that will be compiled into `libblocks` library:
* `checkpoints.dat` contains mainnet checkpoints
* `stagenet_blocks.dat` contains stagenet checkpoints
* `testnet_blocks.dat` contains testnet checkpoints

`daemon` and `blockchain-import` link `libblocks` and will initialize `cryptonote::core` class instance with checkpoints data provided at runtime through `cryptonote::core::init` call.
