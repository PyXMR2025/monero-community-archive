## stmax82 | 2020-11-14T23:23:50+00:00
> I saw this translation (Drittanbieter-Node) being used on Weblate and I thought it was technically correct, it made sense to me.

* "Remote-Node" = a node run "far away" (not on the same computer)
* "Drittanbieter-Node" = "third party node" = a node run and owned by someone else

Every "third party node" is a "remote node", but not every "remote node" is a "third party node", since many users run their own "remote nodes" (which is even recommended since it improves security).

Therefore the translation as "Drittanbieter-Node" ("third party node") seems incorrect.

I still haven't come up with a better translation though...

* "remote" = "fern"
* "remote control" = "Fernsteuerung"
* "remote maintenance" = "Fernwartung"
* "remote node" = ?

> Does it make sense to specify that the wallet connects itself to the node as you did with adding "sich"?

In German yes, in English no. :)

"a client connects to a server" would be "Ein Client verbindet sich zu einem Server".. doesn't work without "sich".

You could also translate it as "Ein Client baut eine Verbindung zu einem Server auf".. but that'd rather be "A client establishes a connection to a server" in English.

> And I used "Software" throughout the German translation since it's pretty much used,

I guess it depends on the context - "software" and "program" cannot be interchanged in many cases.

Wikipedia on "computer program": "A collection of computer programs, libraries, and related data are referred to as software."

> but I can remove "piece of"/"Teil einer" if it's better?

First what I had in mind is Wikipedia's definition for "daemon":

"a daemon is a computer program that runs as a background process".

(short and simple.. it's the whole program, not part of it.. also note that it uses "program" instead of "software")

But after reading this paragraph again I'd actually keep your translation ("Teil einer Software"), because it is more detailed and technically correct:

A program can spawn one (or even more) daemons that keep running in the background, while the same program can provide other functions that run in the foreground, depending on how the program is parameterized. So the daemon actually is only part of the program.
