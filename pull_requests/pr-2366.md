# Original Description
As per #2065:

This optionally allows the transaction key to be deterministically generated in the following way: for a transaction with `N` inputs where key images are `I_1, I_2, ..., I_N`, the transaction secret key `r` is derived as

    r = Hs(b || I_1 || I_2 || ... || I_N)

where `b` is the spend secret key.
