# Original Description
## Summary
- add `atomic` to the Android ARMv7 protobuf try-compile link libraries
- add the same `atomic` dependency to the final Trezor target for Android ARMv7

## Rationale
Recent protobuf/Abseil builds can require compiler-rt atomic builtins on 32-bit Android ARM. The existing Android Trezor path already links `log` in both the protobuf try-compile and the final target, but it does not carry `atomic`, so the configure-time probe and/or final target can fail on ARMv7 while other Android ABIs keep working.

This keeps the extra dependency scoped to Android ARM/ARMv7 and mirrors it in both places where the protobuf/Trezor link happens.
