# Original Description
Regression added in #10396 broke `--data-dir` for usages where the `--regtest` option wasn't added to the variables map. This is most of the blockchain utilities. They don't even work when `--data-dir` isn't specified by the user.

This PR adds `cryptonote::arg_regtest_on` to the blockchain utilities options so that they work again. It also dedups the CLI argument -> network type logic.
