# Discussion History
## vtnerd | 2020-01-31T02:56:13+00:00
I don't see why this matters ... ? Is this purely for aesthetics? The `++current_index >= m_seed_nodes.size()` resets to `0` either way and the conditional check doesn't change.

## moneromooo-monero | 2020-01-31T15:20:00+00:00
The ++current_index >= m_seed_nodes.size() condition would not fire as the call to append_net_address may add to m_seed_nodes. So the current_index does not get set to 0. Since it was walking through the seeds in random order, it needs to get set to the end of the array to walk through the newly added seed nodes. Due to the ++ to check for loop end, setting it to m_seed_nodes.size() would cause the first newly added seed node to be skipped.
