# Original Description
While building Monero 0.18.4.1 from the official release tarball (without Git submodules), the build fails with:

`ld: cannot find -lwallet-crypto: No such file or directory`

This happens because the target wallet-crypto does not exist in the tarball or build system. However, the correct static library that is built is libcncrypto.a.

The file src/device/CMakeLists.txt incorrectly links against wallet-crypto. To fix the issue, replace the reference with cncrypto:

`sed -i 's/wallet-crypto/cncrypto/' src/device/CMakeLists.txt`

This is required for successful builds using the release tarball without relying on Git submodules. Please fix this in the next tagged release to avoid breaking downstream packaging.

Patch is attached. Tested and confirmed working on Gentoo and macaroniOS (cross-compiling with GCC 13.3.0 and Ninja). Please include this fix in future releases to ensure compatibility outside the monero git clone with submodules.

[monero-0.18.4.1-unbundle-dependencies.patch](https://github.com/user-attachments/files/21548110/monero-0.18.4.1-unbundle-dependencies.patch)
