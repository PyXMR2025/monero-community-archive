# Original Description
Adds a new command to the wallet rpc called `--prompt-for-password`.

Current behavior prior to the change is documented here:

> - --wallet-file specified with no password
>   - Asks for password and if valid logins in; failure invalid password message

Now it returns an error, "no password specified; use --prompt-for-password to prompt for a password" in this case.

- --wallet-file specified with no password
  - `--prompt-for-password` not provided? Error message "no password specified; use --prompt-for-password to prompt for a password"
  - `--prompt-for-password` provided, asks for password and if valid logins in; failure invalid password message

Ref: #2179
