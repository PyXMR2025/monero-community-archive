<!-- Describe the pull request in detail. -->
Implements monerod's `zero_commitment` lookup table for common amounts defined here: https://github.com/monero-project/monero/blob/c8214782fb2a769c57382a999eaf099691c836e7/src/ringct/rctOps.cpp#L43

Unlike monerod which uses a simple linear search, this PR implements a binary search over the lookup table, saving a few dozen nanoseconds. 

More micro-optimization could be done like verifying if the amount is over 10 then modulo of amount per 10 should be 0, saving ourselves the lookup cost by directly passing to calculation.

I think you are meant to return here, otherwise this will infinite loop.

As a fun exercise I made a "constant time" algorithm:
```rust
    let log = amount.checked_ilog10().unwrap(); // don't actually unwrap here.
    let div = 10_u64.pow(log);

    let most_significant_digit = amount / div;

    if most_significant_digit * div != amount {
        return H_PRECOMP.vartime_multiscalar_mul([Scalar::from(amount), Scalar::ONE]);
    }

    let index = (most_significant_digit + u64::from(log) * 10 - 1 * (u64::from(log) + 1)) as usize;
    
    ZERO_COMMITMENT_LOOKUP_TABLE[index].1
```

note: this is not actually constant time for cryptographic purposes. it would be good to see how that stacks up.


FWIW: I would rather use rusts build in binary search even if it is a little slower.
