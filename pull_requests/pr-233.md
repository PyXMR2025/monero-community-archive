## hinto-janai | 2024-08-01T21:18:39+00:00
The main trait of this crate, the handler trait.

Handlers are just `tower::Service`s that happen to use our types and implement this `RpcHandler` trait. From there, we can send requests to it, and it will return us a response, which this crate routes back to the client with `axum`.

The request/response/error type are defined in this crate:
- `RpcRequest`
- `RpcResponse`
- `RpcError`

and all handlers must use these types.

## hinto-janai | 2024-08-01T21:22:25+00:00
This is how users get their `axum::Router<H: RpcHandler>`.

This use to just be a free function:
```rust
fn create_router<H: RpcHandler>() -> axum::Router<H> {
    axum::Router::new()
        /* enable every route */
}
```
but I decided to make a builder so users can selectively choose which routes to enable. For `cuprated` purposes, we just use:
```rust
RouterBuilder::new().all().build()
```
but a handler that only expects `/get_blocks.bin` could do:
 ```rust
RouterBuilder::new().binary_get_blocks().build()
```
which would only enable `/get_blocks.bin`.
