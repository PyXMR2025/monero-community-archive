# Original Description
1. Users should be able to assign labels or nicknames to remote nodes
![image](https://user-images.githubusercontent.com/21298211/149438626-3fee11ca-82f4-4811-bf2c-119b546dde71.png)
I would like more than just a port number to differentiate the node network.
2. Remote nodes should be saved for each type of wallet. 
Suppose I have remote nodes X (stagenet), Y (stagenet), and Z (mainnet). I'm currently in a stagenet wallet connected to X. I exit and open mainnet wallet and switch the remote node to Z. I then open up a stagenet wallet. A good UX would connect the stagenet wallet to the remote node X, but currently the behaviour is that it connects to remote node Z. This results in wallet-node mismatch.
