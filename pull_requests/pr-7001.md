## mj-xmr | 2021-02-11T19:12:25+00:00
Since you are touching this line already, it would be a good opportunity to wrap the line after the if () with braces {}, so that the flow isn't changed by accident.

Here's a corresponding clang warning
https://clang.llvm.org/extra/clang-tidy/checks/readability-braces-around-statements.html

Here's some more rationale:
https://www.adam-bien.com/roller/abien/entry/the_only_reason_to_use

"Without curly brackets, you could accidentally write a semicolon after the IF-statements. The semicolon is a valid, empty statement, which will be "execute" instead of the actual (intended) one.

if(condition ); doSomething();"


Another funny accident would occur if somebody comments out the line under if.
