# Original Description
## What
Integrate (un)restricted RPC servers in `cuprated` and some base RPC configuration. No endpoints are enabled (yet).

Default `cuprated` behavior:
- Unrestricted RPC server on `127.0.0.1:18081` (enabled)
- Restricted RPC server on `0.0.0.0:18089` (disabled)

./cuprated --config-file Cuprated.toml

## Panic
`cuprated` will panic if the unrestricted RPC server is binded to a non-local address:
```toml
[rpc.unrestricted]
address = "0.0.0.0:18081"
```
```
INFO Starting RPC server restricted=false address=0.0.0.0:18081
thread 'cuprated-tokio' panicked at binaries/cuprated/src/rpc/server.rs:72:9:
Refusing to start unrestricted RPC on a non-local address (0.0.0.0:18081)
note: run with `RUST_BACKTRACE=1` environment variable to display a backtrace
```
unless the equivalent of `./monerod --confirm-external-bind` is passed:
```toml
[rpc.unrestricted]
address = "0.0.0.0:18081"
i_know_what_im_doing_allow_public_unrestricted_rpc = true
```
```
INFO Starting RPC server restricted=false address=0.0.0.0:18081
WARN Starting unrestricted RPC on non-local address, this is dangerous! address=0.0.0.0:18081
```
The name is stupid but the point is to draw attention to make sure the user knows what they are doing.

`/json_rpc` is technically 1 endpoint - i.e. we either enable all `/json_rpc` methods or none, possible workarounds:

## Return dummy responses
`/json_rpc` can be enabled and "disabled" endpoints will return `Default::default()` responses.

## Return error responses
`/json_rpc` can be enabled and "disabled" endpoints will return errors (in the `tower::Service` impl) which will bubble up to the `axum` router as a `StatusCode` error.

## Return in `/json_rpc` router function
In the `/json_rpc` route fn here:

https://github.com/Cuprate/cuprate/blob/474ff9ed6f474789c32436bc560f866164207bbf/rpc/interface/src/route/json_rpc.rs#L16-L20

do something like:

```rust
match {
    JsonRpcRequest::EnabledMethod1 | JsonRpcRequest::EnabledMethod2 | ... => { ... },
    _ => return Err(StatusCode::501),
}
```
