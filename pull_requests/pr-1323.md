# Original Description
The way the labelbutton was being used was stopping the "resolve" button from showing up when an OpenAlias address was used. Once you got it to show up it would end up replacing the "Address Book" label so this PR changes it to it's own standard button. The regex filter also has to allow '.' and '@' characters otherwise the user can't even enter a proper OpenAlias address
