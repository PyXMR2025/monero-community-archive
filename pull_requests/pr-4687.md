# Original Description
The OSS light-server-wallet will make use of these changes. Please look at the fee estimate one, it differs from the HTTP-JSON-RPC one slightly - it returns `base_fee`, `size_scale`, `fee_mask`, and `hard_fork_version`. The size scale isn't strictly needed, but its somewhat nice to avoid having the clients check version to determine whether the base rate is per-byte or per-kibibyte. If useful, I will update the HTTP-JSON-RPC call eventually.

The output distribution for calls for ZMQ-JSON-RPC and HTTP-JSON-RPC both use the same caching mechanism (including the recent update by @moneromooo-monero ).
