# Original Description
After [this commit](https://github.com/monero-project/monero/commit/237f0179b70eb795772e815f2e9ff6f056f53c22) of [PR #3313](https://github.com/monero-project/monero/pull/3313), to build Monero for Windows in MSYS2, one needs the static ICU libraries as provided by MSYS2/MinGW package `mingw-w64-x86_64-icu`.

A week ago there was a renaming of those static ICU libs, see the corresponding [MSYS2 commit](https://github.com/Alexpux/MINGW-packages/commit/de7b103ac3b53cf9fe9ca7d3afb09ba0394681e9): Starting with version 58.2-3 of the package he letter "s" in the library names was dropped, so e.g. `libsicudt.a` became `libicudt.a`.

This leads to changes in the toplevel `CMakeLists.txt` that has to list those libraries in case of a Windows build.
