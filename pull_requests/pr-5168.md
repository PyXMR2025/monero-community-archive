# Original Description
As dummy payment ID is a legitimate value now (8xzero byte) the `bool get_short_payment_id()` has to be changed a bit as it returns 8x0B also as an indication of failure / absence of short payment ID. 

Because of this construction data does not get updated in `get_construction_data_with_decrypted_short_payment_id()`. As a consequence, the encrypted dummy ID stays in the construction data `extra` and payment ID is scrambled.
