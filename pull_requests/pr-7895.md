# Original Description
[PR 7760](https://github.com/monero-project/monero/pull/7760) fixed a SSL handshake race, but still has exception safety bugs. This brings a simplified fix to the data race and exception+stack reference bug in a smaller PR. One test case is also from the other PR, but perhaps should be committed by the original author.

The two outstanding issues are the same - under pessimistic situations the `async_handshake` hasn't completed/aborted but ownership of the socket remains with the caller. The easiest fix is to pass ownership of the SSL socket to the handshake function, but moving SSL sockets was introduced in Boost 1.74 and our minimum version is 1.56. This may mean heap allocation for the socket object, which is a more invasive change.

The other fix is somehow running `io_service::poll_one` in a destructor call, which is problematic if the object was stopped for some reason.

I've also run these test cases with gcc TSAN without issues.
