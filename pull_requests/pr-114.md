# Original Description
- Adds a `map` module in `cuprate_helper` for mapping data types
- Adds functions for splitting/combining `(u64, u64) <-> u128` which can be used for https://github.com/Cuprate/cuprate/pull/113#issuecomment-2073827676.
- Splits `cuprate_database`'s `cumulative_difficulty` into 2 `u64` bit values

There's a bunch of other mapping functions I'd like to add but will do that later.

A note: although `ExtendedBlockHeader` has `cumulative_difficulty: u128`, it seems to compile fine. What should we do here? Is the overall layout for that type okay and it was only `BlockInfo` that was invalid?
