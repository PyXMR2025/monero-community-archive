# Original Description
### What
Ports all of `monerod`'s binary types at `.bin` endpoints to `cuprate-rpc-types` in the `bin` module. 

Requires https://github.com/Cuprate/cuprate/pull/218.

# Discussion History
## Boog900 | 2024-07-10T18:17:07+00:00
In monerod this is serialized using the macro `KV_SERIALIZE_CONTAINER_POD_AS_BLOB`, which has slightly different behaviour than default. 

Becuase this is a `Vec<[u8; 32>` you would have to use `ByteArrayVec`, for other types like `Vec<u64>` you would have to use `ContainerAsBlob`

## hinto-janai | 2024-07-10T19:53:50+00:00

Either those implement `serde` or we remove it from the macro and add a derive on-top of all types in `{json,other}.rs` manually.
