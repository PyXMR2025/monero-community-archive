# Original Description
Unlike the HTTP RPC method [send_raw_transaction](https://ww.getmonero.org/resources/developer-guides/daemon-rpc.html#send_raw_transaction), which accepts the tx as a hex blob, the equivalent ZMQ method `send_raw_tx` expects a JSON representation of `struct cryptonote::transaction`.

I'd like to add this otherwise-identical `send_raw_tx_hex` ZMQ RPC that accepts the tx in hex (like the HTTP one does).

Also, since the core_rpc_server rejects all txes when not synced, I've added the same check here. The log messages are changed to make it clear they're not coming from core_rpc_server.
