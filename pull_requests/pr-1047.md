# Original Description
This makes `build.sh` work on Fedora 27. Specifically this fixes:
 * Fedora uses the binary name `qmake-qt5`
 * Not all hosts have an `lsb_release` command
 * Disable libunwind on Fedora (like Ubuntu)

With these changes `build.sh` can successfully compile a `monero-wallet-gui` executable. That executable immediately segfaults for me, but that's a pull request for another day.
