# Original Description
The setting for trusting the daemon or not is common to all of the CLI, the RPC and the GUI, so it seems natural to move the boolean variable to wallet2. This also makes the `--untrusted-daemon` flag available to the RPC which was missing previously.

Consequently, the boolean arg `trusted_daemon` to many functions of `wallet2` was dropped. Note, however, that this wasn't applied to `wallet2::refresh()` because the existing code contains a few instances of calling `refresh(false)`.
