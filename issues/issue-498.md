## SyntheticBird45 | 2025-06-03T19:16:52+00:00
I've a hard no for simple declarative description of Cuprate like "A rust monero node.". The project is named Cuprate not monerod-rs. It's boring, while we could have some attractive motto that can be both correct and vehicle some enthusiasm.
Examples i found:

[Zebra](https://github.com/ZcashFoundation/zebra): Zcash - Financial Privacy in Rust 🦓
[reth](https://github.com/paradigmxyz/reth): Modular, contributor-friendly and blazing-fast implementation of the Ethereum protocol, in Rust 
[libbitcoin](https://libbitcoin.dyne.org/): libbitcoin - modular, scalable, async

There are also descriptions that just make use of one word to give some energy or break the formal description:
[clippy](https://github.com/rust-lang/rust-clippy): A **bunch** of lints to catch common mistakes and improve **your** Rust code.
[tlborm](https://github.com/veykril/tlborm): The **Little** Book of Rust Macros

## hinto-janai | 2025-06-03T20:24:01+00:00
I think a simple description like "A rust monero node." is boring as well. I think zebra, reth, and libbitcoin all have descriptive-like titles, none of them are too out there, I think this would be a good middle ground.

> We cannot ethically claim to be "secure".

There's a pattern of less absolutes in security/privacy mottos e.g:

- Qubes OS: A reasonably secure operating system
- PGP: pretty good privacy

We could follow that, although then again, Monero's motto is: "the secure, private, untraceable cryptocurrency".
