# Original Description
Resolves #6619

unbound had a security audit and multiple issues were found and fixed in v1.9.3: https://ostif.org/our-audit-of-unbound-dns-by-x41-d-sec-full-results/

I don't know how relevant they are for monero, still our unbound submodule is unmaintained and outdated. Setting the it to upstream isn't trivial (it would require calling autoconf from cmake). Better solution is to just remove the submodule completely.

Deterministic builds are now using the latest stable unbound version.
