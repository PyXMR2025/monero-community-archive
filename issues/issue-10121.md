# Original Description
`get_blocks_by_height.bin` does not allow requesting pruned blocks and for pruned nodes it will silently drop pruned txs, which for pruned nodes is any V2 or RCT tx as V1 txs are never pruned.  

This issue is to request pruning support for `get_blocks_by_height.bin`.

# Discussion History
## kayabaNerve | 2025-09-28T12:30:25+00:00
`get_blocks_by_height.bin` should likely be removed entirely OR be notably updated to match `get_blocks.bin`. It itself currently has a minimal set of arguments accepted (solely the `heights` array) and a minimal output (only `blocks`, not `output_indices`),

If the numbers are a contiguous range, `get_blocks.bin` is sufficient as the `start_height` field can be specified to a non-zero value and used with the new `max_block_count` field (#9901) to have the desired behavior. If the blocks are unordered, the read will be unordered and the RPC request won't be performant to serve. Since this is a niche RPC call which has fallen through the cracks, I'm not against dropping it in order to encourage RPC requests be made in a way which have performant implementations.
