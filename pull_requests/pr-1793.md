# Original Description
This PR introduces a dedicated merchant page. Current iteration of the merchant features present on the receive page are inadequate to be used as a point of sales system.

The changes in this PR will make it easier for merchants to accept Monero. Run the application in full-screen mode, set the amount to be received and customers scan the QR code.

Alternatively you can provide a payment URL.

Some notes:

- Introduces a menu under Receive called Merchant.
- Optimized for running in full-screen.
- Transaction history shows a 'X time ago' indication. i.e: "the transaction happened 5 minutes ago"
- Incoming transactions are monitored via tx pool.
- For now, clicking on transactions takes you to the History page.
- In the future merchants could optionally specify their to received amount in USD when #1714 is accepted & merged.
