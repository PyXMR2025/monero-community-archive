# Original Description
### What
Closes https://github.com/Cuprate/cuprate/issues/149.

Worth noting this is more of a workaround than a true fix.

### How
This maps `cuprate_database::config::SyncMode::Fast` to `redb::Durability::Eventual` instead of `redb::Durability::None` to prevent the unfreed pages issue.

This means that `redb` will differ slightly compared to `heed`, as it will sync to disk _sometime_ after a write compared to `heed` leaving it up to the OS. This is now documented on the type.
