# Original Description
### What
Moves the `trait Table` generator macro from `cuprate_blockchain` -> `cuprate_database`.

This new `cuprate_database::define_tables` macro:
1. Implements `cuprate_database::Table` on all defined types
2. Creates a `trait {Tables,TablesIter,TablesMut,OpenTables}` specific to those tables

### Why
This allows all crates that build on-top of `cuprate_database` to re-use the macro when defining their tables.
