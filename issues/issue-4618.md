# Original Description
On all of our deployed Monero mainnet instances (3) we're experiencing high CPU load since the update to 0.13.0.2. Load seems to occur in a regular pattern of every 2 hours. Here's a graph:

![image](https://user-images.githubusercontent.com/179920/47016013-cf472280-d14e-11e8-82a9-77b92741b6cf.png)

High spikes reach 100% CPU load. At this stage we're not certain whether it's the RPC wallet or the daemon itself. We can certainly say that mainnet is affected; we're not monitoring the stagenet instances so we don't have any data in this case.

Any ideas what could cause this?
