# Original Description
First stage of a change to ensure excessively large transactions can't be created to clog the mempool.

Stage 1: Remove ability to create transactions > 1.25x block size from wallet

Stage 2: Remove acceptance of transactions > 1.25x block size by transaction pool

Stage 3: Single more sophisticated transaction size checker based on historic median block size, fee priority, and @jollymort's calculations for bumping the median block size by fee
