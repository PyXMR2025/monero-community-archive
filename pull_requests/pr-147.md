# Original Description
Begins implementing `cuprate-rpc-types`.

- Creates macros needed for mass type generation
- Ports over some RPC types (JSON only for now)
- Adds some misc RPC types (e.g. `enum Status`)
- Adds the header/base RPC types (e.g. `struct ResponseBase`)

The main point of this PR is to implement the type generator macro, then following PR(s) will use that macro to port all `{json,bin,other}` definitions.

AFAICT `monerod` puts binary inside JSON strings, e.g. this is the response from `get_txpool_backlog`:
```json
{
  "id": 0,
  "jsonrpc": "2.0",
  "result": {
    "backlog": "�\b����������", // note the quotes, it's a JSON string
    "credits": 0,
    "status": "OK",
    "top_hash": "",
    "untrusted": false
  }
}
```
So I think we just model it as a `String` Or maybe some custom wrapper around `Vec<u8>` so we don't pay UTF-8 costs? The following code works:
```rust
use serde::*;
use serde_json::*;

const JSON: &str = r#"{
  "id": 0,
  "jsonrpc": "2.0",
  "result": {
    "backlog": "�\b����������",
    "credits": 0,
    "status": "OK",
    "top_hash": "",
    "untrusted": false
  }
}"#;

#[derive(Serialize, Deserialize)]
struct GetTxpoolBacklog {
    id: u64,
    jsonrpc: String,
    result: Result,
}

#[derive(Serialize, Deserialize)]
struct Result {
    backlog: String, // a JSON string holding binary data
    status: String,
    untrusted: bool,
}

#[derive(EpeeStuff)]
struct Backlog {/*...*/}

fn main() {
    let s = from_str::<GetTxpoolBacklog>(JSON).unwrap(); // this works

    let epee = epee_decode::<Backlog>(&s.result.backlog.as_bytes());
}
```
