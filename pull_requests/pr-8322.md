# Original Description
As of OpenSSL 3.0, `SHA256_Init`, `SHA256_Update`, and `SHA256_Final` are deprectaed in favor of the higher-level `EVP_*` class of functions. This causes compiler warnings, and sooner or later, will cause build errors as these functions are excluded from distro builds.

This PR does a few things:
* Updates the `sha256sum()` code to comply with OpenSSL 3.0
* Moves `sha256sum` out of `common/util` and into its own aptly named files
* Adds a bunch of error logging
* Uses the system-defined buffer size instead of a hardcoded 4096 value
* Won't throw on file error and functions are now explicitly `noexcept`
* Documentation & extra unit tests
