## SyntheticBird45 | 2024-05-30T21:35:34+00:00
1st: Crates defining Monero intrinsic behavior that is software-agnostic (can be used outside of Cuprate).
3rd: General purpose crates re-implementing monero-related code for Cuprate specifically (usually by redefining API).

```suggestion
| Monero related code re-written for Cuprate purposes   | `cuprate` suffix | `levin-cuprate`, `cryptonight-cuprate`
```

## hinto-janai | 2024-06-13T14:42:05+00:00
A simpler option is to remove prefix/suffixes from directory names and always prefix `cuprate-` to all crate names, like [`reth`](https://github.com/paradigmxyz/reth/tree/main/crates).

E.g.
- `database/` -> `cuprate-database`
-  `levin/` -> `cuprate-levin`
- `epee-encoding/` -> `cuprate-epee-encoding`
- `monero-wire/` -> `cuprate-wire` or `cuprate-monero-wire`

This is clearer if/when uploading to crates.io as well. I think I'd prefer this way actually.
