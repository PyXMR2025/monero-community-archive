Takes too long to fetch DNS checkpoints on startup. 

It will check DNS checkpoints a few moments later in background (from p2p connection thread on a new block/tx).

Will still do the check on startup If a user enforced DNS checkpoints (`--enforce-dns-checkpointing`).
