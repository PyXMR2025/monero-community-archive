# Original Description
The Unbound and OpenSSL depends packags are not reproducible when building android targets because the Android NDK's bundled archiver includes timestamps by default. The default options are not overridden properly for these packages.

The archiver lies about zeroing timestamps by default.

>  [D]          - use zero for timestamps and uids/gids (default)

This PR sets the `ARFLAGS` for these packages to the flags defined in the hosts file, [which includes `D` for android](https://github.com/monero-project/monero/blob/master/contrib/depends/hosts/android.mk#L13).

Note for OpenSSL: it only reads `ARFLAGS` (not `AR_FLAGS`) when passed as an argument.
