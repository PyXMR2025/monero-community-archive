# Original Description
This PR fixes a number of small inconsistencies in the gitian build documentation, reduces redundancy and adds more info where needed.

* Add info on installing docker and creating the `docker` group.
* Fixes usage of git tags to have a real tag example and make sure that the full tag (eg `v0.14.1.0`) is used consistently rather than the various mixed-use before.
* Simplify signing instructions and explain how to verify checksums.
* Many small fixes to ensure consistent file naming in `.assert` and `.sig` files.
* Remove `export` of bash shell variables only used in current shell.
