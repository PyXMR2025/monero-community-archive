# Original Description
Looking at some occurrences of 'key' in translations/monero_core.ts, which I find confusing:

"Secret view key"
"Public view key"
"Secret spend key"
"Public spend key"

"View key (private)"
"Spend key (private)"

I have yet to understand how there can be both a secret and a public spend key, but the issue is mixing "private" and "secret". Use only one!

Personally I'd root for "Public vs Private" as this pair is in line with accepted terminology in crypto.
