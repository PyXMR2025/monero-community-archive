# Original Description
Speed up the daemon startup, ~15-20sec gain.

1. On-demand seed nodes initialization (DNS seed nodes lookup with fallback to hardcoded list). Shouldn't do this on each daemon startup, but only when we have no white peers and want to connect to a seed node.
Run the task in background, i.e. in p2p connection thread on first `connect_to_seed()` call.
2. `m_seed_nodes` access should be thread-safe
3. `m_fallback_seed_nodes_added` flag should be atomic
