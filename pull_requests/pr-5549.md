## jagerman | 2019-08-13T23:20:21+00:00
This `alt_toggle` logic doesn't seem optimal.

The vast majority of the time this gets called to evaluate an alt chain block that will have the same seed height as the current main chain, but with this being `== 0` and the `rx_s_toggle` used by the main chain code path being set using `!= 0` we almost always end up with two identical caches being allocated in `rx_s[0]` and `rx_s[1]` as soon as an alt chain with the same seed height gets hashed.
