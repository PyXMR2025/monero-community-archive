## vtnerd | 2018-12-07T21:59:55+00:00
References are not needed for either argument, and should likely be removed. The penalty, on most ABIs, is copying 2 additional 64-bit values to registers. Those values are immediately retrieved through the reference anyway.

The output should also _probably_ be fixed to `uint8_t` since this function is outputting fixed 0-255 binary values. The one complication will be `parse_hexstr_to_binbuff` but I doubt anything other than `CharT == char` is used in that case. The output of that function should likely be fixed too (always be `std::string`).

## vtnerd | 2018-12-07T22:02:20+00:00
Why is this casting to `(char*)`? I think this is because `.data()` only returns `const char*` in C++11 mode, which makes this undefined behavior (that luckily works almost always). This likely works because I doubt `std::wstring` is ever passed as an argument, and your testing platform(s) don't use reference counted strings anymore. `&res[0]` has to be used here, and avoid C-casting because it masks these fundamental issues.
