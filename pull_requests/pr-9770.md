# Original Description
2048 and 8192 are an unnecessarily low bottleneck for initial chain sync. Noticable if blocks grow (2MB/s is very low) and if using a [dynamic block sync size](https://github.com/monero-project/monero/pull/9494).

I presume FCMP++ will also benefit from higher speeds.

note: a lot of configs floating around already recommend bumping to 1GB/s. This only bumps by 4x to 8192/32768.
