# Original Description
I have tested this on my local node, but it could use some more testing before merge (this PR is to v0.17 branch). The buffer limit for p2p payloads is reduced to 256 KiB until a handshake is requested (outgoing) or successfully processed (incoming).

This places a tighter binding between the context and the levin code. Previously, the context was only required to have `boost::uuids::uuid m_connection_id` as a member.
