# Original Description
The base class `device` already declares the variable `mode`. By redeclaring the same member in `device_ledger` class, the base class' (protected) `mode` is being shadowed and it's not certain whether the `device_ledger` intends to modify (or use) its own instance or the base's one. What is sure, is that it doesn't use the base's one.

For the reason that I can't test this, I'm setting the branch as a draft. The PR does correct the mistake. The question is, whether some functionality isn't broken, because of previous reliance on faulty assumptions.
