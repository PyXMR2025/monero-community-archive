# Original Description
Unsure if suggestions/requests are welcome, but figured why not.  Would it be possible to implement some type of listening support for a server to broadcast a MODT or live announcement?  For instance, the server could broadcast a message on a certain port, that port is specified in config.json then periodically xmrig pings the port to relay any messages to the command window. "We're changing ports on July 1, please update to 12345." It would require implementation on the user end, but could be useful for server announcements.
