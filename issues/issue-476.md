# Original Description
## What
Re-writing the [architecture book](https://architecture.cuprate.org) into a reference book, similar in style and functionality as the [Rust reference](https://doc.rust-lang.org/stable/reference).

## Why
The architecture book is currently written as a learning resource with non-contributors in mind; the style is closer to the [Rust book](https://doc.rust-lang.org/book) than the reference. This is fine although it is:
- costly to maintain
- overly verbose for contributors
- targets a small set of theoretical people

The main purposes of an internal book for the project is to:

- informally describe the various models, services, and internals of the codebase
- write down any notable things that should not be forgotten
- do the above in a precise manner

Switching from the current style to a reference style will be more useful to active contributors.
