# Original Description
### How to use

1. Run `./monero-wallet-rpc --testnet --rpc-bind-port=12345 --wallet-dir=/path/to/dir --rpc-login=test:test --http-wallet`
2. Open http://localhost:12345/wallet in your browser. If using a chromium-based browser (Chrome/Edge/Vivaldi/Brave/...), you can start it in the app mode using `--app=http://localhost:12345/wallet` for better visuals.
3. Enter the RPC username and password (test/test).
4. Open or create a wallet.

![Wallet screenshot](https://i.ibb.co/kJKBrXw/httpwallet2.png)

The wallet offers only the most basic functionality at the moment: opening/creating a wallet, balance, sync progress, creating accounts/subaddresses, sending to a single destination and limited transaction history.

### Implementation details

Changes to existing code are minimal. One small change was needed in wallet RPC method `get_height`, which now also returns the daemon height.

The HTTP wallet itself consists of 5 static files:
* index.html
* styles.css
* wallet.js
* icon.png
* logo.png

These files are served under the virtual directory `/wallet`. The wallet must be explicitly enabled with the `--http-wallet` command line option and requires `--wallet-dir` to be set.

### Security

The wallet is sent to the browser with the `Content-Security-Policy: default-src 'self';` header. This means that the wallet can only comunicate with `monero-wallet-rpc` and should be otherwise completely isolated from the internet.
