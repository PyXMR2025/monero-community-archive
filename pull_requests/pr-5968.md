## moneromooo-monero | 2019-10-24T09:35:34+00:00
The point of the check is to avoid getting the same tx many times from different peers and ending up verifying them all. This does not happen now since there is an early out of the tx is already on the chain or in the txpool, but with deferment it won't be in either yet. So the check needs to be here or you could get a DoS where one peer sends a tx many times.

I think the best might be to always add the tx if not defering (which happens either on a timer or if a block was received). In the first case, no txes are actually added. In the second case, an attacker would have to mine a block first. Then the assertion below still holds.

Does this seem like a good solution to you ?
