# Original Description
This is for [issue 7004](https://github.com/monero-project/monero/issues/7004). The changes the `peer_id` system to be used for IP p2p connections only, and not for I2P/Tor. The fallout:

  * Misconfigurations with `--anonymous-inbound` can result in I2P/Tor self-connections.
  * I2P/Tor could make an outbound connection when an inbound connection to the same node already exists.

The second bullet-point must be an accepted downside, because you cannot have privacy and tracking at the same time.

I decided against a per-connection `peer_id` because it still be used to identify users if an aggressive spy appeared. Take the `peer_id` received from a peer, and try to make some connections to `.i2p` and `.onion` addresses to see which one rejected the connection. This attack cannot be completely removed if the `peer_id` system was still be used for I2P/Tor. Dropping the use of `peer_id` was mitigated by checking for self address connections, leaving only misconfiguration (bullet point 1) as the potential issue.

Testers for this would be welcome too.
