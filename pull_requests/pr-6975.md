# Original Description
The new tool (utils/health/valgrind-tests.sh) is able to run Valgrind checks (cachegrind, callgrind and memcheck) on a list of executables (and their arguments) provided line-by-line via a text file, being the parameter of the script.
The *.out files can be analyzed with the kcachegrind tool.

I have minimized all of the repetitions. This was somewhat harder than expected, because the memcheck tool has a bit different interface than the other tools.

Below are the reports generated for the unit_tests executable:
[Valgrind reports for unit tests](https://github.com/monero-project/monero/files/5483682/valgrind-output.zip)
Summary:
- there were no memory leaks (!) (however, this doesn't mean, that an ever extending vector can't consume all the memory, before the daemon is closed)
- there's an apparent bottleneck within the epee's rolling median's insert method 

![monero-rolling-median-bottleneck](https://user-images.githubusercontent.com/63722585/98035119-4d4ef280-1e18-11eb-9ab7-d718ed461030.png)

Active work time 4.2h
Report generation and trials: 1.5 day (think in terms of power costs, rather than work)
