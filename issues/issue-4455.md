# Original Description
v0.13 has introduced a new field ```device_name``` in the keys file. It is set to "Ledger" for Ledger-backed keys.

So opening a legacy (v0.12) Ledger wallets fails with
```
ERROR	default	src/device/device.cpp:73	Device not found in registry: ''. Known devices: 
ERROR	default	src/device/device.cpp:75	 - Ledger
ERROR	default	src/device/device.cpp:75	 - default
```

Hardcoding ```m_device_name="Ledger"``` in an else block at https://github.com/monero-project/monero/blob/8bf5a00564ad0e86b4698008418e28036013ef9f/src/wallet/wallet2.cpp#L3285-L3288 seems to "fix" the problem.

As I don't understand the purpose of this new field I don't know how to fix this correctly. @ph4r05 ?

# Discussion History
## ph4r05 | 2018-09-26T21:51:37+00:00
I am sorry to hear that, my fault. I will take a look. Could you please try `--hw-device Ledger` parameter as a workaround?

The CLI argument has a precedence over the device name stored in the key file. Once you open the wallet, the key file can be re-generated by calling `password` in the wallet. The new password can be the same as you used before. After the password change the device name is stored to the freshly generated wallet keys file so you can use the wallet as before. 

The device name is required to support multiple different hardware wallets. The device name can also contain the device path.

## ph4r05 | 2018-09-26T23:09:18+00:00
@m2049r So the workaround:

- Start wallet with `--hw-device Ledger`. You can use the wallet file with this argument as normally.
- Regenerate wallet key file by changing the password. Type `password`.
- Exit the wallet and reopen. The device name is stored, argument is not required anymore.
