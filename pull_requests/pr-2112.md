# Original Description
**Allow result submission to origin daemon with self-select**
(_Please see discussion in #2040_).

With `self-select` mode enabled, the `submit-to-origin` config option will let the `SelfSelectClient` submit the solution to both the origin daemon where it got the template from as well as to the connected pool.
Submission to the origin daemon will only be done if achieved difficulty is equal to or higher than the target difficulty received with the block template.

This will enable pool mining for Monero in conjunction with solo merged mining with an altcoin.

Thank you and special credit to @StriderDM (https://github.com/StriderDM)!

------------------------------------

This version of XMRig has been tested in conjunction with [Tari merge mining](https://github.com/tari-project/tari/blob/development/README.md#tari-merge-mining) and Monero mainnet.

Example config.json file:
```
{
    "autosave": true,
    "cpu": true,
    "opencl": false,
    "cuda": false,
    "pools": [
        {
            "coin": "monero",
            "algo": "rx/0",
            "url": "cryptonote.social:5555",
            "user": "<wallet address>.<handle at cryptonote.social>",
            "pass": "start_diff=100000;payment_scheme=pprop;donate=0.5",
            "tls": false,
            "keepalive": true,
            "nicehash": false,
            "self-select": "127.0.0.1:7878",
            "submit-to-origin": true
        }
    ]
}
```
