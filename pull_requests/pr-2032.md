## moneromooo-monero | 2017-05-17T19:47:13+00:00
Can you explain why this change relates to debug builds ?

## stoffu | 2017-05-17T22:54:08+00:00
Sure. Debug build (i.e. shared libs build) for libeasylogging requires the symbol `el::base::elStorage` (aka `ELPP`) to be found when building, which is only defined in mlog.cpp in libepee (which depends on libeasylogging) through the macro `INITIALIZE_EASYLOGGINGPP`. By removing this macro in mlog.cpp and defining `AUTO_INITIALIZE_EASYLOGGINGPP`, that symbol gets defined automatically in easylogging++.cc, hence libeasylogging now builds fine.
