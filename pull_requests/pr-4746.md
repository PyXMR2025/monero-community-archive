# Original Description
hi,

this is the on_restore_deterministic_wallet RPC call for restoring a wallet with a given seed. optionally specify restore_height, seed_offset.

takes all arguments of monero-wallet-cli when starting with --restore-deterministic-wallet:
* seed
* seed_offset
* password
* restore_height
* filename (wallet name)

build and tested successfully on Ubuntu 16.04.

also hopefully satisfies #4058

## stoffu | 2018-10-29T12:24:33+00:00
The overall logic regarding the seed language interpretation seems a bit wrong/pointless. First of all, `old_language` is never set to empty when `words_to_bytes` succeeds, so this branch `if (mnemonic_language.empty() && std::find(language_list.begin(), language_list.end(), m_mnemonic_language) != language_list.end())` seems pointless (also `m_mnemonic_language` is not assigned any value). Whether the given seed words are in the deprecated format or not is determined by either 1) it doesn't have the 25th checksum word, or 2) it's in the EnglishOld language. In the latter case, the name for the new seed language is to be specified via `req.language` (this is done interactively in simplewallet). So it's wrong to return `false` when `was_deprecated_wallet` is true; the process should continue either when 1) the detected language is not the EnglishOld, or 2) the req.language is set to appropriate value.
