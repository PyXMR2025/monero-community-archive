Ah, I see. The copy to the wallet is done because the wallet can't display the error as sent by the daemon because the daemon might include stuff designed to "social engineer" the wallet user (eg, "Error, your wallet is too old, please download the update from $URL"). So the wallet has to recreate the message from the flags the daemon sets in its response.
If you're asking why the daemon keeps the text parts, it's because it's useful in logs when not in an adversarial setup.
