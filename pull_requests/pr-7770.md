# Original Description
As a follow up of https://github.com/monero-project/monero/pull/7768 , I wanted to separate Boost compatibility test from Ubuntu 18.04 backward compatibility test.

It was decided, that in order to provide the required flexibility, the Boost library shall be compiled from source. This is at the same time the only option, that allows to test against the bleeding edge Boost versions, which serves as a great prediction of future incompatibilities, when these Boost versions become distributed by default.

## Findings:
- Ubuntu 20.04 (a.k.a. "latest" at the time of being) is able to compile the latest Boost version (1.76), as well as the minimum officially supported version (1.58), so building the older Boost versions under Ubuntu 18.04 isn't necessary.
- Ubuntu 20.04's `OpenSSL 1.1.1f` refuses to link against Boost of version less than 1.62 (1).
- Ubuntu 18.04 is being employed, to test if < 1.62 can be used, ideally with the minimum supported version 1.58, but it fails, giving a similar error (2), this time for 18.04's `OpenSSL 1.1.1`.

## Conclusions:
- A properly patched, and still supported Ubuntu version imposes a minimum Boost version of 1.62. This has to be reflected in near future.
- Ubuntu 18.04 doesn't need to be tested for Boost compatibility, as 20.04 does the same job already.

## Decisions:
- Shall use Ubuntu 20.04 to compile the latest Boost version from source
- Ubuntu 18.04 doesn't need to be employed
- We shall not automatically test for the lower bound of the Boost library version, in order to conserve resources.
