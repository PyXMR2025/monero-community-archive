# Original Description
## Summary
- When `monerod` is installed as a Windows service using `--install-service`, the service binary path registered with the Windows Service Control Manager was not quoted.
- If the executable path contains spaces (e.g. `C:\Program Files\Monero\monerod.exe`), Windows SCM fails to parse the binary path correctly, causing the service to fail to start.
- This fix wraps the executable path in double quotes when constructing the `full_command` string passed to `CreateService`, ensuring the path is correctly interpreted by the Windows Service Control Manager regardless of spaces in the path.

Ref: [Win32 CreateService docs](https://learn.microsoft.com/en-us/windows/win32/api/winsvc/nf-winsvc-createservicea) — *"If the path contains a space, it must be quoted so that it is correctly interpreted."*
