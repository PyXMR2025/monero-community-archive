# Original Description
`get_transaction_hash` throws when passed a pruned tx, which borks the scanner if tx extra is in unsupported format.

To keep this log as is, we could instead pass the `tx_hash` into `view_incoming_scan_transaction`, but that seemed a little excessive to do here.

Also a passing note (as a reminder for both of us): the scanner cannot throw with any consensus valid tx, otherwise it would be a DoS. Something to keep in mind upon a final review pass.
