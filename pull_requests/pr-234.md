# Original Description
updated dynamic span PR, built on top of on #222 and #223

This is the last piece of the puzzle to contain the runaway spans. Notably, it changes the behavior of queue_proceed, and upgrades the span_limit portion to be dynamically calculated.

The dynamic span limit approximates how many spans can be synced in N minutes, default 2. Includes a new flag: `--span-limit=<minutes>`. There is a soft 100mb restriction (can be modified with pre-existing flag `--block-download-max-size=<bytes>`). The span and size limits can be exceeded temporarily during bursts or if the next span to be synced is missing, but aside from that, should be contained.
