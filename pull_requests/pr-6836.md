## iamamyth | 2020-09-22T20:19:12+00:00
The way this test is written makes it difficult to distinguish timeouts waiting for the initial block versus subsequent ones. It's possible that the timeouts you see are actually for the subsequent blocks, and you should be changing `timeout = 5`, below, rather than this startup timeout. I also think the code would be much clearer (and easier to diagnose failures) if the init and subsequent block cases were separated; for example:
```
for _ in range(60):
    time.sleep(1)
    height = daemon.get_info().height
    if height > prev_height:
        break
else:
    assert False, 'Failed to mine initial block'

target_height = prev_height + 5
while height < target_height:
    cur_height = height
    for _ in range(5):
        time.sleep(1)
        height = daemon.get_info().height
        if height > cur_height:
            break
    else:
        assert False, 'Failed to mine successor to block %d' % (height,)
```

## iamamyth | 2020-09-22T20:27:13+00:00
Alternately, you could keep the single loop and just refactor it a bit, which may be superior:
```
target_height = prev_height + 5
height = prev_height
timeout = 60
while height < target_height:
    seen_height = height
    for _ in range(timeout):
        time.sleep(1)
        height = daemon.get_info().height
        if height > seen_height:
            break
    else:
        assert False, 'Failed to mine successor to block %d (initial block = %d)' % (seen_height, prev_height)
    timeout = 5
```
