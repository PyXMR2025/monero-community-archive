# Original Description
While copying my `data dir` to another drive, I missed copying the rpc_ssl.key file b/c of the file permissions. This change will give a much more clear, descriptive error in that scenario. Before the error message was something like `use_private_key_file: asio ssl error`. I put in a couple of layers of redundancy in case this scenario originates from somewhere that isn't `core_rpc_server`
