## hinto-janai | 2024-04-06T00:04:12+00:00
@Boog900 @SyntheticBird45 

Here's the current pattern for functions in `ops/` - each function comes in 3 flavors:
- `fn()` -> inputs/outputs 1 thing (opens tables for caller)
- `fn_bulk()` -> inputs/outputs N things (opens tables for caller)
- `fn_inner()` -> is the actual implementation (takes in already opened tables)

For example: `get_block()`, `get_block_bulk()`, `get_block_inner()`.

Some open questions:
1. Should `fn()` just be replaced with `_inner()` - as in, should we make the _caller_ open the tables and pass them instead? It would rid the need of an `_inner()`
2. Should these `_inner()` functions be public? Here's an example of what it would look like:

![image](https://github.com/Cuprate/cuprate/assets/101352116/9164964e-122b-4692-b0f0-165945828a80)

## hinto-janai | 2024-04-06T00:25:45+00:00
IMO I think the answer to 1 should be yes - we should take the already opened tables instead of `EnvInner + Tx`.

- The function only takes what it needs
- More flexible for callers
