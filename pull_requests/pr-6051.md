# Original Description
Currently if a user restarts already synced daemon with `--bootstrap-daemon-address auto --no-sync` options, the daemon will immediately disable bootstrapping cause local blockchain is synced.
But it shouldn't do that if `--no-sync` is enabled.

Reported by @selsta
