# Original Description
Adds a regression test network type to allow easier creation of functional and integration tests.

Some notes:

- on_generateblocks RPC call combines functionality from the on_getblocktemplate and on_submitblock RPC calls to allow rapid block creation. Difficulty is set permanently to 1 for regtest.
- In the future, many conditional statements can potentially be replaced by an abstracted class of the networktype, similar to how Bitcoin does this. [See Bitcoin's CChainParams class](https://github.com/bitcoin/bitcoin/blob/master/src/chainparams.cpp)
- I added two small functional tests to give an initial impression of future tests.  [See Bitcoin's functional tests](https://github.com/bitcoin/bitcoin/tree/master/test/functional)

*** edit june 4th 2018 ***

Noteworthy regtest changes include, in order of perceived importance: 
- forced difficulty
- Empty seed peers list
- A low or custom RECENT_OUTPUT_BLOCKS amount to enable transacting at low height

I thought it would be a nice design choice, but for now I have to agree that the following may be better:
- A new forced_difficulty flag
- Functional tests need to use --offline or --add-exclusive-node
- A new recent_output_blocks flag. Making this separate flag seems more clear than making RECENT_OUTPUT_BLOCKS dependent on the forced_difficulty flag.

If agreed, I'll incorporate the flags and undo a lot of changes. I can continue on functional tests in a new PR.
