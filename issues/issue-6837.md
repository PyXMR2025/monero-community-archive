# Original Description
# Overview

When creating an integrated address a 16-byte payment ID is created, and when looking up this payment ID through the `payments` command, an error message will be returned stating that no payments have been found with the given payment id, except the payment ID that is returned has been right padded. 

I've tried this across a variety of different integrated addresses, verified that the transaction shows up in my outbound transactions, and that the transaction can be found on public block explorers. I suspect the problem is because the payment id that is looked up, has right padded 0's.

* [Transaction on the testnet](https://testnet.xmrchain.net/search?value=5fa93c5c40e9841fe0982cecf921a1a187a89ca2e32a9d2feffb9912e5a5f174)
* [Payments output](https://gist.github.com/bonedaddy/6cecad4166d5305e6687268d848f0c0a)

* Monero WalletRPC: v0.17.0.0-release
* Monero Network Used: testnet
