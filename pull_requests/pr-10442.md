# Original Description
- created a custom RPC client wrapper (t_rpc_client) for a Monero-based daemon
- reworked RPC handling into a strongly typed system using templates (T_req, T_res)
- introduced a unified response structure:
  - rpc_result<T>
  - includes OK, BUSY, NETWORK_ERROR, RPC_ERROR, TIMEOUT

---

retry + reliability layer
- added a retry system (execute_with_retry)
- implemented:
  - max retry limits
  - exponential backoff
  - jitter (randomized delay)
- built automatic handling for:
  - daemon BUSY responses
  - transient network failures
  - timeout retries

---

RPC abstraction layer
- wrapped both RPC styles:
  - /json_rpc (JSON-RPC calls)
  - REST-style HTTP RPC calls
- standardised request flow:
  - request struct > HTTP call > response struct > rpc_result
