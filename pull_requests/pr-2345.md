@moneromooo-monero This is not exactly what was discussed in https://github.com/monero-project/monero/pull/2332 or IRC, but is slightly closer to what you were trying to do except with a golang style type-erasure technique.

```c++
struct custom_address {
    constexpr static bool equal(const custom_address&) noexcept { return false; }
    constexpr static bool less(const custom_address&) noexcept { return false; }
    constexpr static bool is_same_host(const custom_address&) noexcept { return false; }
    constexpr static bool is_loopback() noexcept { return false; }
    constexpr static bool is_local() noexcept { return false; }
    static std::string str() { return {}; }
    static std::string host_str() { return {}; }
    constexpr static uint8_t get_type_id() noexcept { return uint8_t(-1); }
};
```

is an example of a class that almost meets the requirements (`equal` and `less` don't do what they are supposed to do). `network_address` now automagically makes it work with hopefully minimal overhead (less construction overhead, but slightly more comparison overhead due to generic nature).
