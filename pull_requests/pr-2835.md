# Original Description
First, there was a bug, libzmq-dev wasn't installed, so it couldn't even build at all.

Then, the original docker file only build `master` branch of the repository. you couldn't build any other branch.

it was a bit wasteful, as it was git clone'ing the repository in docker container. which shouldn't be a necessity, as the host that trigger the build had very likely a local checkout of the repository anyway.

I fixed all these, and I propose to switch to multi stage build. it simplify a few things (no need to cleanup).
and switch to an entrypoint instead of a cmd, to let users pass it some additional parameter, one of which can be logging level
