# Original Description
This PR adds a new fuzzer in the tests/fuzz/fuzz_rpc directory and updates the CMakeLists.txt accordingly. The fuzzer is built in two versions, each targeting a different set of RPC endpoint functions in the src/rpc directory.

## DavidKorczynski | 2025-08-14T22:25:12+00:00
The `FuzzedDataProvider.h` is actually part of the LLVM Framework and libFuzzer. We added this purely because of passing the CI tests here. The fuzzers are meant to run by way of OSS-Fuzz (https://github.com/google/oss-fuzz/tree/master/projects/monero) and `FuzzedDataProvider.h` is already provided by the libraries/compiler in their build environment (this Dockerfile https://github.com/google/oss-fuzz/blob/master/projects/monero/Dockerfile). As such, I don't think we should adjust the `FuzzedDataProvider.h` simply because it's just LLVM code we have here to support the CI.

For what it's worth, we don't need this file in this manner when building/running in the OSS-Fuzz environment (it's only for satisfying Monero's CI). Originally we downloaded it dynamically (https://github.com/monero-project/monero/pull/10004#discussion_r2228501179).

Am not sure what the preference is here? We could remove it and try to avoid building the fuzzer in the CI -- that would probably makes things a bit cleaner. Am not sure if this fuzzer will ever be built outside the OSS-Fuzz environment.
