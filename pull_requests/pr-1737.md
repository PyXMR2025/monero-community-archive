## vtnerd | 2017-02-19T03:06:23+00:00
How about moving the code below to this break statement? All branches `return`, and would remove the need for a temporary variable (and memory allocation/copy) to hold that path that matched.

```c++
std::ifstream power_stream;
for (const std::string& path : POWER_SUPPLY_STATUS_PATHS)
{
  if (is_file_exist(path))
  {
    power_stream.open(path);
    if (power_stream.fail())
    {
      LOG_ERROR("failed to open '" << path << "'");
      return boost::logic::tribool(boost::logic::indeterminate);
    }
    return boost::logic::tribool( (power_stream.get() != '1') );
  }
}
LOG_ERROR("Couldn't find battery/power status file, can't determine if plugged in!");
return boost::logic::tribool(boost::logic::indeterminate);
```
Moving the `std::ifstream` above the loop prevents multiple construction/destruction too. I know creating `std::ostream` are usally expensive to create due to locale synchronization, but I cannot recall the penalty for `std::ifstream`. Don't see a downside though.
