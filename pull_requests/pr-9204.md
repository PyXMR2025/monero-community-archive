# Original Description
The `native_libtapi` depends package is not reproducible because it embeds the checked out git hash of whatever git repository it happens to have been extracted to, which is that of the monero repo in our case. If the `native_libtapi` package is cached, runs for different monero commits that otherwise do not invalidate the cache will use a `native_libtapi` package with a hash that does not match the hash of a clean build.

This does not make its way into release binaries, but hinders automated reproducibility scanning.
