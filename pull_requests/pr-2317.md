# Original Description
The [first commit](https://github.com/monero-project/monero/commit/71fbcf7996e06465b5baa9453b346039d878e536) imports `amd64-51-30k` and `amd64-64-24k` from [`supercop-20170725/crypto_sign/ed25519`](http://bench.cr.yp.to/supercop/supercop-20170725.tar.xz). I encourage @iamsmooth @moneromooo-monero @hyc @fluffypony @luigi1111 @kenshi84 to verify that the code is the same from that source. 

The [next commit](https://github.com/monero-project/monero/commit/674c7e7bf33e60a1b880532c365e819ed946e473) (ping @hyc @luigi1111) updates the supercop code to be compliant with `-fPIC` or `-fPIE`. Some platforms (OSX for example) compile in this mode by default, and I do not see a good way to detect whether position independent code is being used to select a non-position independent variant.

The [last commit](https://github.com/monero-project/monero/pull/2317/commits/8f38b22252accb018d956fa6dc5cd2a9bd715560) are the changes to the monero baseline to integrate the ASM code into the wallet (@hyc and @luigi1111  might want to peek at the `*choose_tp.s` files adapted from supercop code to include `z` field). The current ref10 implementation is not modified, and this code is **optionally** used for wallet scanning only. See associated [readme](https://github.com/vtnerd/monero/blob/8f38b22252accb018d956fa6dc5cd2a9bd715560/src/wallet/crypto/README.md). @iamsmooth @luigi1111 @hyc @moneromooo-monero ... should this be enabled by default? Is it worth the extra maintenance to even use this code? The _bulk_ of this code is upstream from supercop, and now written by me.

## Performance Numbers (OSX on Mac mini mobile i5)
Note that the `monerod` system in my test setup should have slightly higher than normal latency. There was still a ~8min improvement. A much larger drop in user CPU time was seen. Improvements to the fetching code should help. The `amd64-51-30k` version always fetched more blocks.
