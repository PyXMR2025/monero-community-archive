## j-berman | 2022-08-17T06:59:45+00:00
Shouldn't this "Nothing requested" be the valid incremental case as mentioned in the comment below?

In any case, I have an alternative suggested matrix of cases:

```
if (all)
{
  if (count == 0 && start == 0)
    export all

  else if (count == 0 && start > 0)
    export all from start

  else if (count > 0 && start == 0)
    export count from 0

  else if (count > 0 && start > 0)
    export count from start
}
else if (!all)
{
  if (count == 0 && start == 0)
    export all from first output with m_key_image_known && !m_key_image_request
    
  else if (count == 0 && start > 0)
    export all from first output with m_key_image_known && !m_key_image_request, after start
    
  else if (count > 0 && start == 0)
    export count from first output with m_key_image_known && !m_key_image_request

  else if (count > 0 && start > 0)
    export count from first output with m_key_image_known && !m_key_image_request, after start
}
```

Biggest reason for this being that it supports the old behavior exactly as it was when `count == 0 && start == 0`. I also think each case has a valid use
