# Original Description
I added this to be able to send transfers with unlock_time from the wallet:

`locked_transfer [<mixin_count>] <addr> <amount> <lockblocks> [<payment_id>]`

`lockblocks` is the number of blocks to lock the transaction for. I set this to max 1 million blocks now, about 4 years, figured people don't want to lock up their transactions for 100 years by accident. It only works for blocks, no time stamps, and only 1 amount and address.

It asks for confirmation before sending the transaction:

```
Transaction fee is 0.023000000000, of which 0.003000000000 is dust from change.
The unlock time is approximately 221.20 days.
Is this okay?  (Y/Yes/N/No)n
```

I'm not sure how popular unlock_time is, but I can see people want to use it as a way to lock up monero savings and not panic sell with large swings in prices.
