## stoffu | 2018-11-30T04:01:46+00:00
A while back, the wallet API function `createWalletFromKeys` changed from the old insecure version which doesn't take the password string as parameter to the current secure version which takes the password as parameter. When I added this additional kdf round parameter to the wallet API, I didn't bother updating the old deprecated functions. This part of the GUI happened to be using the deprecated version, so I had to substitute the empty string as the password.
