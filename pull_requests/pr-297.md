# Original Description
When opening an existing database, don't set LMDB map size to be the default initial size for a new database. While it's not persisted to disk (if a larger map size was set in the past), it will cause the running instance's map size to be the current data.mdb file size. For sparse files, or those which otherwise don't have the size pre-allocated, this leads to a map size that won't allow new writes.

Check if resize is needed at startup.

Include specific database error in more error messages.
