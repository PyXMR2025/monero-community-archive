## hinto-janai | 2024-02-26T21:20:37+00:00
Sorry, these docs were a little misleading...

This variant will just open LMDB with [`MDB_NOSYNC`](http://www.lmdb.tech/doc/group__mdb__env.html#ga5791dd1adb09123f82dd1f331209e12e) and never call `sync()`. This matches `fastest:async` mode in `monerod`. The readers will still be able to see writes regardless of this, it's referring to _disk_ syncs, not the database data itself so:

```rust
db.put("key", value);
db.get("key"); // <- will get value even with `Fast`, probably from memory directly
```
