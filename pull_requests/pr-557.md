# Original Description
This commit corrects pluralization of terms in documentation tables by removing unnecessary apostrophes.  It adjusts table column widths as necessary to account for reduced character counts in the longest entry of a column, where necessary.

# Discussion History
## ndorf | 2021-03-15T19:12:08+00:00
I think this apostrophe style was an attempt, perhaps unsuccessful, to refer the reader to things explicitly defined in the [Types section of the same document](https://github.com/monero-project/meta/blob/9afcf4328fb6de30adb039fab86ac9c38400939a/api/lightwallet_rest.md#types). I guess nothing is really lost by simply dropping them in the other instances, but I'm not sure about this one, with the suffix change?

Is there a better way we could handle this? Hyperlinks to the corresponding type definitions would be ideal when viewed with a markdown viewer, but would make the plain text form harder to read. Perhaps something like "array of `binary` objects"? Or maybe I'm overthinking this, and this PR is good as-is? Any thoughts from @vtnerd, @moneroexamples?
