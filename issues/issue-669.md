Looks the two threads are failing when outputing log messages via macros like _info_c(). Removing all log statements from saveToFile(), save(), and lambda in dataLogger() and ~dataLogger() does prevent the segfault.

Not sure, but maybe `cLogger` (an object with static lifetime: `cLogger gCurrentLogger`) gets destroyed before `data_logger` (a singleton created on demand, stored in a `static unique_ptr` field of a class). Probably worth a try to create `cLogger` explicitly, and keep it as a singleton, similarly to `data_logger`, instead of declaring it as a static object and hoping the init/destroy order will be correct.
