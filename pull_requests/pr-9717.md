# Original Description
Remove unused Rangeproof types `RangeProofBulletproof `, and `RangeProofMultiOutputBulletproof`. In the prove code, if the `RangeProofBulletproof` enum was passed, then one bulletproof was created per output. If `RangeProofMultiOutputBulletproof` was passed, then K bulletproofs were produced, where K is the number of one bits in the count of the outputs, Neither of these are allowed by consensus: https://github.com/monero-project/monero/blob/2226d4f7e0bf316aa0d4241b9b498f928add4b8b/src/cryptonote_basic/cryptonote_format_utils.cpp#L150-L154 and https://github.com/monero-project/monero/blob/2226d4f7e0bf316aa0d4241b9b498f928add4b8b/src/cryptonote_basic/cryptonote_format_utils.cpp#L174-L178.

Also fixes an actual bug with the boost serialization of type `tools::wallet2::signed_tx_set`. If the serialized payload has a version < 4, then the range proof type is assumed to be `RangeProofBulletproof`, which is incorrect.
