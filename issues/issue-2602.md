# Original Description
Current: 
![image](https://user-images.githubusercontent.com/45968869/70869898-37b8ac00-1f6c-11ea-9ac8-62ea922ea833.png)

Proposed: 
![image](https://user-images.githubusercontent.com/45968869/71422307-9ca48e00-265f-11ea-9db4-8d5252061714.png)

My suggestions:
* Change expand/colapse icons from + and - to ^ and V (up and down arrows, the same used in Advanced options)
* Change text to `Description (optional)`
* Change Description field placeholder to: "_Only seen by you_". It's not necessary to inform the user that it will be saved to local wallet history. Current placeholder "_Saved to local wallet history_" doesn't inform who will receive or will be able to see the description (only the sender vs. both sender and receiver). 

There is potential for confusion when using the current placeholder:
- Payment ID (now deprecated) was a text field that was sent to the receiver
- Traditional banks have a `Payment reference` field that is sent to the receiver
- Some payment services have a `Description` field that is sent to the receiver. See below:

![image](https://user-images.githubusercontent.com/45968869/70869890-22dc1880-1f6c-11ea-8353-8ff95d5510c6.png)
