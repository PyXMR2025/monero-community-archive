# Original Description
Our pool is encountering error -4 "original_output_index too large" when sending out payments via the <code>transfer_split</code> RPC. The transaction contains 32 addresses, without payment ids or integrated addresses.

The (my) pool code creating the transaction and which worked flawless up until now is here:  https://github.com/coinfoundry/miningcore/blob/dev/src/MiningCore/Blockchain/Monero/MoneroPayoutHandler.cs#L136

This is a duplicate of #2472 but we are running v0.11.1.0 which is supposed to contain a fix for this. So there must be something else triggering the bug not covered by the fix.
