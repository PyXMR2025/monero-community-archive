# Original Description
This includes a saner check that FCMP++ compatible wallets *must* point to FCMP++ compatible daemons even before the fork. Recall they must do so in order to sync the tree.

Note: we should expect **all** FCMP++ compatible daemons to return `daemon_hard_forks` to the `get_version` RPC, so we can get rid of all the logic handling the case when it's empty.
