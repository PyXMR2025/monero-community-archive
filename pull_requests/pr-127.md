# Original Description
Reported by @nahuhh:

> The migration log also prints a bit funny

```
1703300 / 3390087 blocks (50% of step 2/2)  /2)  
```

> The second /2)

_____

This is caused by the first `std::cout` that prints when setting up locked outputs having more chars than the second `std::cout` that prints when building the tree.

I fix it by making sure to include space chars that are wider than the max print would be for each `std::cout`, and flushing the line. Can see a similar technique to clear a line used [here](https://github.com/monero-project/monero/blob/8e9ab9677f90492bca3c7555a246f2a8677bd570/src/common/util.cpp#L907).
