## SChernykh | 2025-07-07T19:20:28+00:00
What's the difference between `peer.adr.is_same_host(cntxt.m_remote_address)` and `peer.adr == cntxt.m_remote_address`? You now have both variants in the same conditional expression.

## vtnerd | 2025-07-07T20:02:15+00:00
`is_same_host` checks only the host portion - the port is skipped. The `operator==` does a host+port check.

We want a host only check for this new portion because it needs to match on incoming connections where the port is typically randomly chosen. This will block connections if two `monerod` are using the same ip with the same randomly chosen 64bit id value.

The other `operator==` check is for anonymity networks where only outbound connections have an identifier. This allows for 2 connections (one in each direction) when each side is running standard code. I don't think this can be improved as the inbound identity is intentionally random over these networks.
