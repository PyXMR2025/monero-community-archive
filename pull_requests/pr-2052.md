# Original Description
This PR adds support for getting information about memory modules as requested in #2038 for Windows, Linux, FreeBSD and macOS. Information about memory timings is not available yet since it requires different platform dependent mechanisms to access it. 

#### 3 ways to to access DMI data:
1. Basic information printed in the miner log on startup.
2. New miner API endpoint `GET /2/dmi`.
3. Online benchmark, DMI data is already supported by the Cloud API and later will be displayed on a benchmark page too.

#### 2 ways to disable DMI data:
1. Compile time: new CMake option `-DWITH_DMI=OFF`.
2. Runtime: new command line option `--no-dmi` or config option `"dmi": false,`.
