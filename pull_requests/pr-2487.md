# Original Description
~~(This PR builds on top of PR #2436)~~

The following conversation in #monero on September 19 inspired this PR:
```
<azy:> can you prove that you have received a payment?
<moneromooo:> Yes, see check_tx_proof IIRC. Or check_tx_key, an alternative.
<azy:> moneromooo, that's proving you've made a payment, not proving you've received one
<hyc:> if a transaction has been sent, then nothing can prevent it from being received
<hyc:> sent/mined/confirmed
<hyc:> who would you ever need to prove receipt to?
<azy:> the world
<Bish:> the world is not enough
<moneromooo:> Good point. To prove you received a payment, you could give out your view secret key. Not sure whether there's anything better though.
<moneromooo:> Not sure if you get to know the tx secret key as a recipient. If you do, the same system can be used. I don't thikn you do though...
```

So, this PR adds another way of proving that the prover has received some funds in a given tx by revealing the shared secret `D=a*R` where `a` is the view secret key and `R` is the tx public key; i.e. the signature consists of `D` and a pair of scalars `(c,s)` that proves `D=a*R` and `A=a*G`.

The command `get_tx_proof` is replaced by a pair of commands `get_tx_proof_out` which is identical to the previous one and `get_tx_proof_in` which is newly introduced here. Example:
