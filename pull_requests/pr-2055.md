# Original Description
Implements `miner::get_system_times`, `miner::get_process_time` and `miner::on_battery_power` for OSX so that background mining works on OSX.

## moneromooo-monero | 2017-05-30T08:23:45+00:00

## jtgrassie | 2017-05-30T10:33:10+00:00
Yes this should work according to Apples docs because frameworks symbols are marked as weakly linked which means at runtime the function will either be resolved or null, hence the null check. 

Unfortunately I do not have access to a machine running < 10.7 so I cannot properly test this though. 

The other option is to use Apples "availability macros" at compile time but this would mean binaries compiled on < 10.7 would not have this feature if run on 10.7+ machines.
