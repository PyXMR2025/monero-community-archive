# Original Description
This improves block and transaction blob/binary to C/C++ structure performance by ~60% (in some cases, see below). The changes are pretty basic, but touches 27 files in some of the most critical areas of of code. The read performance is already fairly quick, so this might be overzealous. These functions will show up in profiles at times because of the frequency of their use (basically, everytime a block or transaction needs to be "read"). As an example, responding to a p2p download request improves by a tiny ~1.6% on a Ryzen 3 CPU. 

The bulk of the gains come from dropping a copy of the payload before reading. The remainder comes from eliminating virtual methods calls and some `std::istream` stuff. There is also a difficult to measure benefit with CPU cache usage reduction.

Unpacking 100 blocks (2253800-2253799)
      CPU/Build             |  Microseconds  | Difference
------------------------------|----------------------|---------------
Ryzen 3900x / PR       |            86          |    +0%
i7-7Y75 / PR                |            91          |    +5%
Ryzen 3900x / master |            205         |   +135%
i7-7Y75 / master          |            231         |   +168% (+153% relative)

Unpacking 4,224 transactions from above blocks
      CPU/Build             |  Microseconds  | Difference
------------------------------|----------------------|---------------
Ryzen 3900x / PR       |          7,820       |   +0%
i7-7Y75 / PR                |        10,343       |   +32%
Ryzen 3900x / master |        15,623       |   +100%
i7-7Y75 / master          |         25,481      |   +226% (+146% relative)

One thing to note is that the fanless laptop CPU is now +32% from a Ryzen 3900x, instead of the current +63%. My expectation is that the improvements will have more benefits to smaller cache CPUs (amongst other less sophisticated internals).
