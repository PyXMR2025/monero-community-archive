# Original Description
Sorry for the churn. I intended to hoard this until the [code in my recent issue](https://github.com/monero-project/monero/issues/6406) was PR'ed, but its probably a better idea to push this concept out now inside of letting it rot. The first commit is from PR 6350, with the second commit being the new code.

This creates a class that rapidjson can use to write directly to a buffer that can be passed into ZMQ without any copy. This same class can be used by the epee binary output code (with nearly zero changes), but code expecting `std::string`s need to change. Epee JSON output can also be replaced, but that requires a few `operator<<` functions. See comments in `byte_stream.h` for why this is an improvement over `rapidjson::StringBuffer` and `std::ostringstream`.
