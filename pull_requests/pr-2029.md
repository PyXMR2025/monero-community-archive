# Original Description
Adds support for the `monero://` scheme. allowing the GUI to launch from a (but not limited to) a browser.

Following parameters are supported:

- `monero://<address>`
- `monero://<address>?tx_amount=5.0`
- `monero://<address>?tx_description=foo`
- `monero://<address>?tx_payment_id=foo`
- `monero://<address>?tx_amount=5.0&tx_description=foo&tx_payment_id=foo`

This is also what MyMonero supports; [link](https://github.com/mymonero/mymonero-app-js/blob/fe8028556d4582d1e747e52871e068fd19b886ee/local_modules/MoneroUtils/monero_requestURI_utils.js#L61)

IPC stands for inter process communication. We use [QLocalServer](https://doc.qt.io/qt-5/qlocalserver.html) to communicate URI commands whenever an instance of the GUI is already running in order to avoid starting the GUI each time an user clicks on a monero link.
