# Original Description
There is a `boost::uuids::to_string` method since Boost 1.45 and an `operator<<` has long since existed, making these to-strings unnecessary. Moving the other function drops one include from a commonly used header. This also moves things one step closer to removing `boost::lexical` from the header too (which brings in lots of headers itself).
