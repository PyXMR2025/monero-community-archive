# Original Description
The epee's string_tools.h had 2 main problems: inline functions and a dependency on boost_lexical_cast, which is heavy enough. Everything that could have been un-inlined, is now in a .cpp file, and the 2 templates depending on boost_lexical_cast are now extracted. It turned out, that not all dependees needed both headers at the same time.
After this has been done, I have figured out, that the boost_lexical cast is still heavily used through other headers and thus I decided to get rid of these instances in one go.
