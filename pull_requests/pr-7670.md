The archive is a .tar file, stored uncompressed. While downloading it, GitHub zips it on the fly to a .zip, which reduces the size 4 times. Since we run the job on Ubuntu anyway, wouldn't it make sense to store it compressed as a tar.gz file in the first place?

If you agree, then the suggested change would be:

```
export OUTPUT_TAR="$VERSION.tar"
export OUTPUT="$OUTPUT_TAR.gz"
echo "OUTPUT=$OUTPUT" >> $GITHUB_ENV
/home/runner/.local/bin/git-archive-all --prefix "$VERSION/" --force-submodules "$OUTPUT_TAR"
tar -czvf "$OUTPUT" "$OUTPUT_TAR"
```
