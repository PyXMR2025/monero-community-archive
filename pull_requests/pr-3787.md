# Original Description
  ## What
  This PR hardens stratum networking paths and fixes several hidden reliability issues:

  - Replaced unsafe `uint64_t*` casts with `memcpy`-based reads to avoid unaligned/aliasing UB.
  - Added fallback handling for partial `uv_try_write` writes using `uv_write`.
  - Added transport cleanup (`resetTransports`) to prevent reconnect-path leaks (`Socks5`/`TLS` objects).
  - Added async write callbacks to release write batons and close on write failure.

  ## Why
  These paths could cause:
  - architecture-dependent crashes or wrong behavior (UB),
  - unnecessary disconnects on partial writes,
  - memory/resource leaks during repeated reconnects.

  ## Scope
  Key files:
  - `src/base/net/stratum/Client.cpp`
  - `src/base/net/stratum/Client.h`
  - `src/base/net/stratum/DaemonClient.cpp`
  - `src/base/net/stratum/DaemonClient.h`
  - `src/base/net/stratum/EthStratumClient.cpp`
  - `src/net/JobResult.h`
  - `src/net/JobResults.cpp`

  ## Validation
  - Static checks with `rg` confirmed unsafe patterns were removed in modified paths.
  - CMake configuration was attempted; full compile/run validation is pending toolchain availability.

  ## Notes
  - Local/generated artifacts were removed from tracking (e.g. `build-*`).
