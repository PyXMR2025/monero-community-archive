# Original Description
Previously `Wallet::multisigSignData` method could not properly serialize transaction with more than 2 signatures. In other words transaction signature with 3 or more signatures required didn't work with wallet api.

This happened because of using wrong `wallet2::save_multisig_tx` method overload.
