# Original Description
(I made this PR based on a request by @xeyleukfnd - he/she may want to answer any questions/comments from the community.)

This patch adds `--generate-from-spend-key <filename>` option to `monero-wallet-cli`. The functionality is practically the same as `--generate-from-keys`, with the difference being that the CLI doesn't ask for the public address and the view secret key which are both automatically derived from the spend secret key (assuming the wallet being deterministic). The main motivation is that sometimes users might want to store the spend secret key instead of the seed for some reason (maybe @xeyleukfnd can explain more). The functionality of deriving the public address and the view secret key from the spend secret key are already available in https://xmr.llcoins.net/addressgen.html anyway, so it seems reasonable to support this functionality officially in the CLI.
