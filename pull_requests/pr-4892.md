# Original Description
Since #4795 was merged, AppleClang (both 8.0 and 9.0) emits 173 warnings of the following kind:

> rctOps.cpp:45:22: warning: suggest braces around initialization of subobject [-Wmissing-braces]

https://build.getmonero.org/builders/monero-static-osx-10.12/builds/5588/steps/compile/logs/warnings%20%28338%29

Seems like this is the compiler's bug (https://stackoverflow.com/questions/31555584/why-is-clang-warning-suggest-braces-around-initialization-of-subobject-wmis), and GCC fixed it by not including `-Wmissing-braces` in `-Wall`.
