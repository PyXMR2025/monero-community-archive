# Discussion History
## moneromooo-monero | 2019-04-05T10:19:07+00:00
The default goes to trusting any CA. Is that intended ?

## vtnerd | 2019-04-07T17:31:37+00:00
Yes. The choices are between no verification, the somewhat crummy system "root" CA system, or user provided information (when can be a onion/i2p address). I set this as the default, since this is "standard" SSL.

As per our discussion, if the user enables SSL via command line or the new `set_daemon` RPC, then a certificate, fingerprint or i2p/onion address must be provided.

## moneromooo-monero | 2019-04-07T21:28:10+00:00
use_proxy will also be true if using a plaintext proxy, no ?

## vtnerd | 2019-04-08T00:53:10+00:00
The connection between wallet and proxy is always plaintext. This is attempting to force the usage of SSL through the proxy to verify the endpoint.
