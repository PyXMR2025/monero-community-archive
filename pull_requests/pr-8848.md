# Original Description
I'm not sure what the point of this arg is.

- On *nix, despite what the help text claims, it does not return the OS for which the binary was built. Instead, it returns information about the system it is currently running on. Just run `uname -a` or `file monerod` yourself?

- On windows, it return whichever compatibility mode the executable is running under, which does not seem useful either. Also, `get_windows_version_display_string` is just an extra maintenance burden.
