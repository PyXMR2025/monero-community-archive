# Original Description
- These Makefile targets predate `depends` and are not static and not cross-compiled.
- `make depends` should be used for cross-compiled static builds.

# Discussion History
## jeffro256 | 2025-02-13T23:27:20+00:00
Why was this section removed ? 

## iamamyth | 2025-02-13T23:33:35+00:00
I originally had the same thought, but, if you look below, `make release-static` is now the default build instruction, so this is redundant.

## tobtoht | 2025-02-14T09:16:23+00:00
I removed this section because it is misleading. `release-static` will not produce statically-linked binaries on Linux. To statically link dependencies (other than libc) on Linux, `make depends` should be used, which is documented under the "Cross compilation" section.

Static builds for Linux are [not yet](https://github.com/monero-project/monero/blob/master/CMakeLists.txt#L1012) supported, however work is underway (#9207).
