## Tzadiko | 2025-03-19T21:40:11+00:00
It gives the developer clarity into what the variable is referring to. Ask yourself what is clearer:

```cpp
const bool allow_pruning = false;
if (!method(..., allow_pruning))
```
or
```
if (!method(..., false)) // Says nothing about what is being set to false.
```
which is why some developers prefer to leave a comment.

Speaking of that, you may want to provide a descriptive name instead of a comment. Up to you.
