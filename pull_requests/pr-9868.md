# Original Description
Addresses [issue #9866 ](https://github.com/monero-project/monero/issues/9866)

- None of the methods used by other parts of this repository were changed (e.g. `getAll()` in `TransactionInfo` is unused). 
- There are no heap allocations anymore, and no virtual table pointers, decreasing the size of the objects being used. 
- `getAll` returns a const-reference instead of a copy.
- Impl data members were moved into the non-Impl base class, with the Impl class being erased.
- There is also this funky use of `friend` in C++ that is incorrect (in my opinion). For instance, a class being its own friend. Many classes being friends of Wallet, just to get access to m_wallet.
- Move TransactionInfo into vector, instead of copying.
- Remove const from TrasnsactionInfo::Transfer, which implicitly deletes the objects move constructor.

I can keep going (e.g. PendingTransactionImpl, UnsignedTransactionImpl), but I don't want to if there is push-back against this. I don't see why there would be though, given the agreement that this needs to be cleaned here: https://github.com/monero-project/monero/issues/9461

Built the GUI to see if something broke and it didn't break the TransactionHistory. Accompanying GUI PR: https://github.com/monero-project/monero-gui/pull/4425

## SChernykh | 2025-03-27T15:01:30+00:00
`TransactionInfo` has an `std::set` and `std::vector` inside it, so it will be better to move-construct it here (and in other places) instead of copying `ti` (push_back makes a copy) and then destroying `ti`.
```
m_history.emplace_back(std::move(ti));
```
will be better.

## Tzadiko | 2025-03-27T15:18:44+00:00
Thanks for pointing this out. I changed it to `m_history.push_back(std::move(ti))` which has the same behavior as emplace_back in this case. I usually lean on emplace_back when Im forwarding the arguments of the object to be created in-place (instead of passing in the object itself). For example, I made the move to emplace_back here: https://github.com/monero-project/monero/pull/9868/files#diff-1c5bf7c1b219b1deddcabb7b4d67e8bca138de9ec3be7fa6a7d46609933e8fc4R200
