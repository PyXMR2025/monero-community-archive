# Original Description
Tor/i2p nodes will sometimes send peer lists of size > the expected max to outgoing connections, which causes tor/i2p connections to drop. This PR ensures the peer list remains <= the max size.

I have a decent degree of confidence this is a significant cause of tor/i2p dropped connection issues in #6938 and #6631 (and sporadic failed tx submissions on tor nodes like in #8251), but it's not the only cause of these issues. Still in the process of investigating and patching what look like clearly apparent bugs, separate from this one.

In testing on my machine, I observed that my tor node will push peer lists larger than the max size on average every 1-3 minutes or so (sometimes the gaps are larger, sometimes smaller),  which is followed by the existing outgoing tor connection dropping. Here's what I believe is happening under the hood:

1. OUTGOING tor node runs [peer_sync_idle_maker](https://github.com/monero-project/monero/blob/9f814edbd78c70c70b814ca934c1ddef58768262/src/p2p/net_node.inl#L2004) in the idle loop
2. In 1-3 minute intervals, the OUTGOING node calls [do_peer_timed_sync](https://github.com/monero-project/monero/blob/9f814edbd78c70c70b814ca934c1ddef58768262/src/p2p/net_node.inl#L2117), which makes the [COMMAND_TIMED_SYNC](https://github.com/monero-project/monero/blob/9f814edbd78c70c70b814ca934c1ddef58768262/src/p2p/net_node.inl#L1231) request to MY tor node
3. MY tor node responds with peer list of size > expected max in [handle_timed_sync](https://github.com/monero-project/monero/blob/9f814edbd78c70c70b814ca934c1ddef58768262/src/p2p/net_node.inl#L2486-L2487), **because it adds my own .onion to the peer list even though the list is already at max** (the bug)
4. OUTGOING node [processes the response](https://github.com/monero-project/monero/blob/9f814edbd78c70c70b814ca934c1ddef58768262/src/p2p/net_node.inl#L1161) and sees that the peer list is too large in [handle_remote_peerlist](https://github.com/monero-project/monero/blob/9f814edbd78c70c70b814ca934c1ddef58768262/src/p2p/net_node.inl#L2161-L2165), then closes the connection

This PR's fix makes sure MY node won't response with a peer list larger than the max when adding my own .onion to the peer list in step 3.
