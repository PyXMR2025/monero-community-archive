# Original Description
~~Package archives are not uploaded yet to https://downloads.getmonero.org/depends-sources~~ done

I attempted to edit `fetch_file` so that `$(package)_file_name` is always used for the fallback download.

# Discussion History
## mj-xmr | 2021-07-07T11:32:58+00:00
Could you explain why this change and the changes in the packages were needed?

## selsta | 2021-07-07T15:30:12+00:00
Some download file names only contain the version, e.g. `contrib/depends/packages/eudev.mk` download file name is `v3.2.6.tar.gz`, which can clash with other packages and also isn't self describing, so it gets renamed to `eudev-v3.2.6.tar.gz` after download.

The fallback mirror has `eudev-v3.2.6.tar.gz` uploaded so the call for `fetch_file_inner` has to be updated in a way that the download file name is never used.
