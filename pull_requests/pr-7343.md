# Original Description
`m_deletion_initiated` in the levin code doesn't appear to be needed. The value is always checked after `start_outer_call()` which (semi-secretely) does a `weak_ptr`->`shared_ptr` in `connection::add_ref()`. The object is never destructed if a `shared_ptr` is held, so the atomic value is never going to be `true` in the checks.
