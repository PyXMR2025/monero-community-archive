# Original Description
Moves some hash function domain separators to a common configuration location, to reduce the likelihood of future collision.

# Discussion History
## iamamyth | 2020-02-13T22:14:44+00:00
Although the config value happens to match `sizeof(rct::key)`, this seems a bit confusing for future readers. I presume the config value is declared an unsigned char array because the `rct::key` type isn't available, so I would suggest a static assertion here that `sizeof(rct::key) == sizeof(config::HASH_KEY_MULTISIG)` in order to clarify the intent and reduce the potential for future accidents (a stray delete in the config could end up copying from uninitialized memory, which may or may not produce a test failure).

## SarangNoether | 2020-02-13T22:16:50+00:00
Good point. Updated.
