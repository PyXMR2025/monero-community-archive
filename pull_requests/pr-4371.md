# Original Description
This is used heavily in the [light wallet server PR](https://github.com/monero-project/monero/pull/4139). It provides a value-or-error object similar to `result<T, E>` in Rust or `expected<T, E>` / `outcome<T, E>` as seen on the boost mailing lists. The behavior in use in the light wallet library:
* `T foo() noexcept;` can never fail.
* `T foo();` only has unrecoverable errors.
* `expect<T> foo() noexcept;` only has recoverable errors.
* `expect<T> foo();` has recoverable and unrecoverable errors.

In other words, throwing an exception is for shutting down the process whereas `expect<T>` is for returning errors that can be handled. The error type is `std::error_code` which is extensible - making it possible to track the source of an error. Example with LMDB errors:

```c++
namespace
{
    struct close_env
    {
        void operator()(MDB_env* ptr) const noexcept
        {
            if (ptr)
                mdb_env_close(ptr);
        }
    };

    using environment = std::unique_ptr<MDB_env, close_env>;

    // Actual function from light wallet server
    expect<environment> open_environment(const char* path, MDB_dbi max_dbs) noexcept
    {
        MONERO_PRECOND(path != nullptr);

        MDB_env* obj = nullptr;
        MONERO_LMDB_CHECK(mdb_env_create(std::addressof(obj)));
        environment out{obj};

        MONERO_LMDB_CHECK(mdb_env_set_maxdbs(out.get(), max_dbs));
        MONERO_LMDB_CHECK(mdb_env_open(out.get(), path, 0, open_flags));
        return out;
    }
}
// Usage:
int main()
{
    expect<environment> maybe_env = open_environment("/path", 20);
    if (maybe_env)
        // do stuff
    else
        std::cout << maybe_env.error().message() << std::endl; // from mdb_strerror

    try
    {
        environment env = open_environment("/path", 20).value(); // throws on error
        env = MONERO_UNWRAP(open_environment("/path, 20)); // add file+line to exception
    }
    catch (std::system_error const& e)
    {
        std::cout << e.what() << std::endl; // from mdb_strerror AND maybe file+line
    }
}
```

std::error_code is lightweight, just an `int` and a pointer. So copying a `expect<int>` requires a 24-bit `memcpy` on 64-bit builds and 12-bit `memcpy` on 32-bit builds.

Tested on OSX and Ubuntu 16.04 with valgrind.
