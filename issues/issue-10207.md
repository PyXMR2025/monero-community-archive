# Original Description
when blocks are popped automatically (during reorgs etc) is understandable that txs would be retained.

When manually popping blocks, txs should be dropped / deleted.

Problem: its extremely inefficient to move txs back to the pool, making large pops essentially impossible

This request is for manually popping blocks to simply delete them, along with their txs, instead of wasting _tons_ of time moving them around.

The majority (or all) of the time that you manually pop blocks, you are trying to roll the chain back to an earlier state for testing or because of an issue. I dont see why we'd retain the txs.
