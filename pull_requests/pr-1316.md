# Original Description
I was asked to move the wallet rpc functionality into a new process, and  this is the result. Very little new code was added, most of it is moved around. I hope this isn't too aggressive, but I know the code well enough to move it around based on requests/suggestions.

- `wallet_args.h/cpp` now handles the log file and concurrency command line arguments + the various global initializations for the two wallet processes.
- Command line argument handling for p2p-daemon and passwords have been moved to `wallet2` so that both processes can use the same code.
- `password_container` was moved to `src/wallet`
- Various fixes for translations
- `monero-wallet-rpc` now asks for wallet password if its not provided, so it does _not_ have to read it from a file or from a command line argument.

Questions:
- Are there any translations to test with?
- Is the `translations/monero.ts` file maintained? Should I revert changes to that file?
