## j-berman | 2026-03-09T17:07:14+00:00
@Boog900 requested the change [here](https://github.com/0xFFFC0000/monero/pull/63#discussion_r2727520783), and raised DoS concerns [here](https://github.com/seraphis-migration/monero/pull/184#discussion_r2701073109). I think the change is a reasonable one because the connection code really isn't designed to write many huge sliced responses to a single request: [this section](https://github.com/monero-project/monero/blob/b9998fc9e1c280d30e4b40ba983b1964eb256d66/contrib/epee/include/net/abstract_tcp_server2.inl#L821-L861) shows that could cause issues by tripping the limiter in place. Missing some txs here seems better than ending up dropping the connection.
