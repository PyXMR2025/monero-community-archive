### What
This adds some types/traits to `cuprate_rpc_types`.

This adds 6 enums for encapsulating RPC requests/responses:
- `crate::json::JsonRpcRequest` & `crate::json::JsonRpcResponse`
- `crate::bin::BinRequest` & `crate::bin::BinResponse`
- `crate::other::OtherRequest` & `crate::other::OtherResponse`

2 traits that contains some metadata about RPC request/responses:
- `RpcCall`
- `RpcCallValue`

and changes to the type generator macro to auto implement the traits.

### Why
Both enum types and metadata is needed for actual RPC usage: https://github.com/Cuprate/cuprate/pull/233.

See below review.

## hinto-janai | 2024-08-01T00:23:00+00:00
This trait is a runtime mirror of `RpcCall`. It exists because sometimes we can only know the RPC call at runtime, for example, we must deserialize methods in `/json_rpc` before we know what it is.

Once we have some method variable, we do:
```rust
async fn route(...) {
    if request.is_restricted() && rpc_server_is_restricted {
        /* ... */
    } else {
        /* ... */
    }
}
```
`RpcCall` is used when we _know_ the type at runtime, e.g. `/get_height` must lead to a `GetHeightRequest` type, so we can use `GetHeightRequest::IS_RESTRICTED` instead.

## hinto-janai | 2024-08-01T00:26:21+00:00
Example of a request enum.

These request/response enums are used within `cuprate_rpc_interface`'s `tower::Service` API, not directly, but something like:
```rust
// inside `cuprate_rpc_interface`
enum RpcRequest {
    JsonRpc(cuprate_json_rpc::Request<JsonRpcRequest>),
    Other(OtherRequest),
    Binary(BinRequest),
}
```
then this broad `RpcRequest` type is used like such: `tower::Service<RpcRequest>`.

## hinto-janai | 2024-08-01T00:28:24+00:00
The way types can easily implement `RpcCall` is by using this syntax when defining a type.

`GetBlockTemplate` is neither empty or restricted so it has nothing. The type generator macro will detect this and implement:
```rust
impl crate::RpcCall for GetBlockTemplate {
    const IS_RESTRICTED: bool = false;
    const IS_EMPTY: bool = false;
}
```
