This would be a replacement for #7306 and #7330.

From IRC:

```
7330 or unmodified code isn't consistent with library api
1. it's expected be heap allocated memory
2. release_field_name returns string pointer that must be freed by caller in general
3. memwipe should be called before release_field_name in general
but it's better to replace that with set_allocated_field_name(new std::string(...)) and use mutable_field_name in order to wipe it after
```

## ph4r05 | 2021-01-20T21:36:28+00:00
As discussed previously, wiping [0] when string is empty is undefined / dangerous.

## selsta | 2021-01-21T11:17:18+00:00
Would like to hear @vtnerd’s source for "The m_body[0] expression is only valid if !m_body.empty()."

## vtnerd | 2021-01-24T01:47:45+00:00
My fault, the [the C++ specification changed](https://en.cppreference.com/w/cpp/string/basic_string/operator_at) since C++11. The expression `m_body[0]` is valid when `m_body.size() == 0` but the value will be `0` and the behavior is undefined if the value is changed. So this expression and the change in the PR are unnecessary since its always setting the value to `0`. The prior C++ spec was stricter but more obvious.

## vtnerd | 2021-01-24T01:48:14+00:00
It's also not inherently wrong to check, because that still meets the requirement for not changing the `NULL` character.
